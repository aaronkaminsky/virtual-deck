### Integration Issues (Tech Debt)

**1. Production host fallback bug** (`usePartySocket.ts:6`)
- Both branches of `VITE_PARTYKIT_HOST ??` ternary resolve to `localhost:1999`
- Safe today because `deploy.yml` injects the env var
- Fix: `import.meta.env.DEV ? 'localhost:1999' : 'virtual-deck.aaronkaminsky.partykit.dev'`
- Affects: ROOM-01, ROOM-02, ROOM-03, ROOM-04

**2. LobbyPanel copy-link inaccessible after connection** (`App.tsx:12`)
- `if (gameState)` renders `BoardDragLayer`; `LobbyPanel` fallback only shown ~100ms pre-connect
- Room URL is visible in browser address bar; requirement is functionally met
- No copy affordance in `BoardView` — UX improvement opportunity

**3. Dead action types** (low severity)
- `DRAW_CARD`: server handler present, never dispatched from UI — CARD-02 satisfied via `MOVE_CARD`
- `SHUFFLE_DECK`: server handler present, never dispatched from UI — CTRL-02 satisfied via `SHUFFLE_PILE`

**4. DECK-03 placeholders** (intentional)
- `CARD_BACK_URL = ''` and `CARD_FACE_URL` returns `''` in `card-art.ts`
- Intentional per Phase 2 design; real art is a code-change swap operation
