  <how-to-verify>
    Setup:
    1. Open two terminals.
    2. Terminal 1: `npm run dev` (PartyKit server)
    3. Terminal 2: `npm run dev:client` (Vite at http://localhost:5173)
    4. Open two browser windows (different `BrowserContext` to simulate two players): `http://localhost:5173` and `http://localhost:5173?player=p2` (or use two browsers / one private window). Join the same room with distinct display names.
    5. From Player 1's hand, drag 3-4 cards to the canvas one at a time so the canvas has some pre-existing cards.

    Selection (MULTI-01) — Player 1:
    - [ ] Click a single canvas card → blue ring (`0 0 0 2px #60a5fa, 0 0 0 4px rgba(96,165,250,0.3)`) appears around it.
    - [ ] Click a second canvas card → both have rings.
    - [ ] Badge `2 selected` (or higher count as more clicks) appears in canvas top-left corner.
    - [ ] Click one of the selected cards again → ring removes; count badge updates or disappears at <2.
    - [ ] Click empty canvas area (not on a card) → all canvas cards deselect; badge hidden.
    - [ ] Select 2 canvas cards → click a hand card → canvas selection clears (zone-exclusive); hand selection ring appears.
    - [ ] Select 2 canvas cards → click a spread zone card (any pile in spread mode) → canvas selection clears; spread selection ring appears.

    Canvas→canvas group drop (MULTI-02 + MULTI-03):
    - [ ] Select 2 canvas cards spaced ~80px apart.
    - [ ] Drag one of them ~150px to the right and down.
    - [ ] During drag: drag handle card visible at 50% opacity in DragOverlay (scale 1.05). Passenger ghost div visible at 50% opacity, tracking the drag. Source cards both at opacity 0 (invisible placeholders).
    - [ ] Release on empty canvas area: both cards land at new positions; relative spacing preserved (still ~80px apart).
    - [ ] Both cards are now visually on top of any pre-existing canvas card (z-order: above).
    - [ ] Internal z-order between the two moved cards is preserved (whichever was on top before is still on top).
    - [ ] Player 2 sees the new positions update in real time within 1-2 seconds.

    Hand→canvas group drop (MULTI-02):
    - [ ] In Player 1's hand, click 3 cards to select them (ring + lift visual matches existing hand pattern).
    - [ ] Drag one selected card from hand to canvas.
    - [ ] During drag: only the drag handle card is visible in DragOverlay (no passenger ghost divs for hand→canvas sources — this is intentional per D-16/D-11; ghost divs require canvas x/y coordinates which hand cards don't have; the DragOverlay handle provides sufficient visual feedback).
    - [ ] On drop: all 3 cards land on the canvas, NOT in the hand. Relative spacing roughly matches their pre-drag arrangement in the hand row.
    - [ ] All 3 cards face-up. All above any pre-existing canvas card in z-order.
    - [ ] Player 2 sees the 3 cards appear on the canvas; Player 1's hand count drops by 3.

    Spread→canvas group drop (MULTI-02):
    - [ ] Move some cards into Player 1's personal spread zone (drag from hand to spread).
    - [ ] Multi-select 2 cards in the spread zone.
    - [ ] Drag one to the canvas; release.
    - [ ] During drag: only the drag handle card visible in DragOverlay (same as hand→canvas; no passenger ghost divs for spread→canvas — intentional per D-16/D-11).
    - [ ] Both spread cards land on canvas; both removed from spread zone; relative offsets preserved.

    Bounds violation (MULTI-04):
    - [ ] Select 2 canvas cards near the right edge of the canvas.
    - [ ] Drag them rightward so that, on release, one card would be outside `canvasW - CARD_W`.
    - [ ] Release: BOTH cards snap back to their pre-drag positions. No card moves. No toast. No red flash. No error highlight. Silent.
    - [ ] Verify the other player did NOT see any movement either.
    - [ ] Repeat with hand→canvas: select 2 hand cards, drag toward right edge of canvas so a passenger would overflow. On release: hand unchanged, no card placed.

    Undo (MULTI-04 atomicity):
    - [ ] Perform a 2-card hand→canvas group drop.
    - [ ] Press undo (existing controls). All 2 cards return to hand in original positions. Canvas reverts to pre-drop state. ONE undo press, not two.
    - [ ] Repeat for canvas→canvas reposition: undo restores both cards to pre-drag x/y/z.

    Selection clears on drop:
    - [ ] After ANY successful group drop, no canvas cards have selection rings; badge gone; clicking other zones works normally (selection cleared on dragEnd per D-06).
    - [ ] After bounds-violation snap-back, selection also clears.
