## Color

All tokens resolved from src/globals.css CSS variables. Values listed are the dark felt theme defaults (`:root` and `.dark` are identical in this codebase).

| Role | CSS Token | Resolved Value | Usage |
|------|-----------|----------------|-------|
| Dominant (60%) | `--background` | hsl(160 38% 16%) | Board surface, middle band, canvas shell background |
| Secondary (30%) | `--card` | hsl(160 38% 10%) | Opponent hand bar (`bg-card`), card pile face |
| Accent (10%) | `--primary` | hsl(38 92% 50%) | Drop-target border highlight on PileZone (`border-primary`), badge background |
| Destructive | `--destructive` | hsl(0 84% 60%) | Destructive actions only — not present in Phase 31 |
| Border / muted | `--border` | hsl(160 20% 20%) | Sidebar `border-r`, PileZone borders, default pile ring |

Accent reserved for: PileZone drop-target border highlight only. The `--ring` token (same amber value) is used for focus rings. No other elements use the accent color in this phase.

**Sidebar divider:** Use `border-r border-border` (Tailwind token `border-border`). This resolves to `hsl(160 20% 20%)` — a 1-pixel muted line, no background color contrast change, consistent with D-04.

**Canvas shell:** `bg-background` — plain felt green. No decoration, no placeholder text, no dashed border (D-05).

## Layout Contract

This phase is primarily a layout restructure. The following rules are binding for the executor.

### Middle Band Structure

```
<div class="flex-1 min-h-0 flex items-start">
  <!-- Sidebar (flex-shrink-0, no scroll) -->
  <div class="flex-shrink-0 flex flex-col gap-2 py-2 px-2 border-r border-border">
    {pilePiles.map(pile => <PileZone ... />)}
  </div>
  <!-- Canvas shell (empty at Phase 31) -->
  <div class="flex-1 min-w-0 overflow-hidden bg-background self-stretch" />
</div>
```

Key layout rules:
- Outer middle band: `items-start` (not `items-center` — D-11). Remove `px-4 gap-4` from outer div.
- Sidebar: `flex-shrink-0` so it never yields space to the canvas (D-01, D-02, D-03).
- Sidebar padding: `py-2 px-2` (8px each side — two pile zones with natural width).
- Sidebar gap: `gap-2` (8px between draw and discard piles).
- Canvas: `flex-1 min-w-0 overflow-hidden` — `overflow-hidden` chosen over `overflow-auto` for Phase 31 empty shell; Phase 32 will decide scroll strategy with full canvas card context (D-06, discretion resolution from RESEARCH.md open question 3).
- Canvas height: `self-stretch` so it fills the full band height without a fixed value (D-06).

### Responsive Behavior

- At 375px viewport: sidebar stays visible at natural PileZone width; canvas receives the remaining horizontal space (D-03). No collapsed state, no overflow scroll on sidebar.
- The outer scroll container (`overflow-x-hidden overflow-y-auto sm:overflow-hidden`) is unchanged.
