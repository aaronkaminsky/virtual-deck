## Overview

Replaced the fixed communal grid with a free-form canvas where each card has an absolute (x, y, z) position and can be placed anywhere, overlapped, selected in groups, and dragged with full real-time sync. Includes a no-card-loss invariant, overlap hit-testing, multi-card group drops with all-or-nothing bounds checking, and mobile edge-pan support.

## Milestone Summary

**Key Decisions:**

- `CanvasCard` as standalone type (`{card, x, y, z}`) rather than extending Pile — keeps all existing MOVE_CARD/UNDO/viewFor handlers untouched; separate `canvasCards[]` field is cleaner
- `CanvasDraggableCard` uses `useDraggable` only (no `useDroppable` per card) — canvas zone is the single droppable; individual cards droppable would create z-order collision complexity
- `PLACE_ON_CANVAS` handles both initial placement and canvas→canvas repositioning — unified single action with `z=max+1` keeps undo symmetric; server computes z to prevent client race conditions
- `viewFor()` broadcasts `canvasCards` without masking — canvas cards are placed face-up by design; no privacy concern
- All-or-nothing bounds rule for group drops (vs anchor-and-clamp) — simplicity; no partial placement
- `scrollOffsetRef` shared between CanvasZone and BoardDragLayer (not state) — live value in drag closure; state would capture stale scroll offset causing off-position drops
- Hard canvas height cap removed in 35-02 — flex-1 distributes height correctly; `max-h-[240px]` caused pile sidebar/canvas height mismatch

**Issues Resolved:**

- dnd-kit ID collision between SortableSpreadCard and DraggableCard — resolved in Phase 31 by removing nested useDraggable inside useSortable
- Double-counted scroll offset in drop math — removed `scrollOffsetRef` dead code in Phase 35 post-CR

**Technical Debt Incurred:**

- Canvas is not scrollable/pannable beyond initial viewport (CANVAS-F01 deferred)
- Card positions do not persist if all players disconnect simultaneously (CANVAS-F02 deferred)
- No formal milestone audit — REQUIREMENTS.md traceability for MULTI-* and MOBILE-* was stale at close; corrected manually
