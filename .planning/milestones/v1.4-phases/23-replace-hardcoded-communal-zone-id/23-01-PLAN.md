The existing drag-reorder caller continues to snapshot (omits the flag).

```
case "REORDER_HAND": {
  const hand = this.gameState.hands[senderToken];
  if (!hand) break;
  const idSet = new Set(hand.map(c => c.id));
  if (
    action.orderedCardIds.length !== hand.length ||
    !action.orderedCardIds.every(id => idSet.has(id))
  ) {
    sender.send(JSON.stringify({ type: "ERROR", code: "INVALID_REORDER", message: "..." } satisfies ServerEvent));
    break;
  }
  takeSnapshot(this.gameState);
  const cardMap = new Map(hand.map(c => [c.id, c]));
  this.gameState.hands[senderToken] = action.orderedCardIds.map(id => cardMap.get(id)!);
  break;
}
```

The plan changes only the `takeSnapshot(this.gameState);` line, wrapping it in `if (!action.skipSnapshot)`.

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-23-01 | Tampering | `REORDER_HAND` server handler — client passes `skipSnapshot: true` to bypass undo recording | accept | `skipSnapshot` only suppresses the local server-side undo entry for the sending player; it does NOT modify other players' state, escape the sender's own hand, or grant new privileges. Worst case: a malicious client makes its own undo stack shallower for itself. No multi-player exploit surface. |
| T-23-02 | Tampering | `REORDER_HAND` server handler — invalid `orderedCardIds` with `skipSnapshot: true` | mitigate | Existing length+membership validation (lines 322–332 of `party/index.ts`) runs BEFORE the snapshot branch, so bogus ids are rejected with `INVALID_REORDER` regardless of `skipSnapshot`. Task 2 includes a regression test for this. |
| T-23-03 | Repudiation | Sort actions silently mutate hand order without an audit trail | accept | Out of scope — Virtual Deck has no audit log model; reveal/reset/etc are all unsnapshotted by design (see `party/index.ts` lines 510, 514). Sort joins this category by design. |
| T-23-SC | Tampering | npm/pip/cargo installs | N/A | No package installs in this plan. Plan does not invoke `npm install`. |

Security posture: No new auth, crypto, or input-handling surface. Existing `INVALID_REORDER` validation guards the `skipSnapshot` path the same way it guards the default path.
