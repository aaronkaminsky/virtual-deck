# Requirements: Virtual Deck

**Defined:** 2026-05-15
**Core Value:** Players can see the shared table and their own private hand update in real time, with no one able to see each other's face-down cards.

## v1.4 Requirements

### Hand Management

- [x] **HAND-01**: Player can toggle their hand face-up to reveal all cards to other players
- [x] **HAND-02**: Player can toggle their hand face-down to re-hide their cards
- [x] **HAND-03**: Hand revealed/hidden state is broadcast in real time to all connected players
- [x] **HAND-04**: Hand revealed state is persisted in server room state so reconnecting players see the correct current state
- [x] **SORT-01**: Player can cycle through hand sort modes: original order, by suit, by rank; sort persisted server-side via REORDER_HAND (does not enter the undo stack)

### Select All

- [x] **SELECT-01**: Player can click a "Select All" control on any pile to select all cards in it
- [x] **SELECT-02**: Player can click a "Select All" control on any spread zone to select all cards in it
- [x] **SELECT-03**: A group selected via "Select All" can be dragged to any valid drop target using existing multi-card drag

### Play Area Grid

- [x] **GRID-01**: Communal spread zone displays as a 2-row fixed grid; cards snap to column positions; multiple cards can stack per cell; player can drag cards between cells

### Layout & Visual Polish

- [x] **POLISH-01**: Empty piles and spread zones display no body text — label above is sufficient
- [x] **POLISH-02**: Pile zone controls appear at the top of each pile, inline with the label
- [x] **POLISH-03**: Personal spread zones are positioned closer to the communal/draw/discard area
- [x] **POLISH-04**: Zone heights and spacing are reduced for a more compact board
- [x] **ZONE-01**: Personal spread zones are hidden when empty; a drop target appears when the player begins dragging a card
