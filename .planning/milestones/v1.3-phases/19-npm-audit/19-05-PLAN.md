Why these exact classes:

- `max-w-[160px]` (default, applies at less than 640px) — at 375px the opponents row already has `flex-1 overflow-x-auto`, so the row supports horizontal scrolling. Bounding each opponent column at 160px means a single column never alone forces page-level overflow. 160px is approximately the natural width of a 5-card opponent hand at 42px-card phone size with overlap, plus a small margin for the connected dot, name, and Badge.
- `sm:max-w-none` (applies at 640px and up) — desktop has plenty of horizontal space; reverting to no max-width preserves existing desktop layout exactly. This is the project's mobile-first responsive convention (CONTEXT.md D-04).
- `overflow-x-hidden` (applies at all sizes) — clips any opponent SpreadZone content that exceeds the column. The opponent SpreadZone already has its own overflow-x-auto, which provides scroll inside the zone; the parent overflow-x-hidden ensures the column boundary is the visual stop. Safe at sm: because the column has no max-width and nothing organically exceeds it.
