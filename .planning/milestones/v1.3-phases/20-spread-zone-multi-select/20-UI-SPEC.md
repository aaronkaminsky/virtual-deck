## Interaction Contract

### Card Selection (SPREAD-01)

Mirror `SortableHandCard` in `HandZone.tsx` (lines 9‚Äì60) exactly. The following specs are the contract:

| Interaction | Behavior |
|-------------|----------|
| Click unselected spread card | Add to selection; apply lift + ring |
| Click selected spread card | Remove from selection |
| Click card in a different zone | Clear current selection; begin new selection in clicked zone (`selectionSource` clears) |
| Escape key | Clear entire `selectedIds` Set (already globally wired in `BoardDragLayer`) |
| Visual lift | `translateY(-6px)` via `resolvedTransform` ‚Äî applied only when `isSelected` and not actively dragging |
| Selection ring | `ring-1 ring-primary/30 ring-offset-1 ring-offset-background rounded-md transition-transform duration-150` |
| Drag-origin placeholder | `absolute inset-0 rounded-md border-2 border-dashed border-muted-foreground` ‚Äî shown when `draggingCardId === card.id` |
| `aria-pressed` | `true` when selected, placed AFTER `{...attributes}` spread to avoid TS2783 dnd-kit override |

**Click vs. drag disambiguation:** Use `didDragRef` pattern ‚Äî `onPointerDown` records start position; if pointer moves before release, treat as drag and suppress click toggle. This matches the existing `HandZone` + `DraggableCard` approach.

### "N selected" Badge (D-03)

| Condition | Behavior |
|-----------|----------|
| `selectedIds.size >= 2` AND this zone is `selectionSource` | Show badge in zone header |
| `selectedIds.size < 2` OR zone is not `selectionSource` | Hide badge |
| Badge appearance | `ml-2 text-xs bg-primary text-primary-foreground rounded-full px-1.5` ‚Äî identical to HandZone line 115 |
| Badge content | `{count} selected` (e.g. "3 selected") |

### Zone Interactivity (D-07, D-08, D-09)

| Zone | `interactive` prop | Drag-from | Drop-to | Selection |
|------|-------------------|-----------|---------|-----------|
| Communal zone (`pile.id === "play"`) | `true` | Yes ‚Äî all players | Yes ‚Äî all players | Yes ‚Äî any player |
| Own personal spread zone | `true` | Yes | Yes | Yes |
| Opponent personal spread zone | `false` | No ‚Äî `SortableSpreadCard` replaced with plain `CardFace`/`CardBack` | Yes | No |

When `interactive={false}`:
- `SortableSpreadCard` is not rendered; plain `CardFace`/`CardBack` renders instead.
- No `useSortable` hook invoked (eliminates drag listeners).
- Drop target (`useDroppable`) remains active ‚Äî opponents can still receive cards.

### Multi-Card Drag (SPREAD-03)

| Behavior | Spec |
|----------|------|
| Trigger | Drag a selected card from a spread zone |
| Effect | Entire `selectedIds` set moves as a group |
| Insert dialog | No dialog ‚Äî always inserts at top of target pile (D-05) |
| Drop targets | Pile (stack), HandZone, another SpreadZone |
| Selection cleared after move | Yes ‚Äî clear `selectedIds` and `selectionSource` on successful `PLAY_CARD_SET` dispatch |
