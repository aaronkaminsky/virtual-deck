#### 4. PARTYKIT_TOKEN Secret Format (Critical)

**Test:** Check the value of the `PARTYKIT_TOKEN` secret. Confirm it is the full contents of `~/.partykit/config.json` (a JSON object, e.g. `{"login":"...","access_token":"..."}`) rather than a bare token string.
**Expected:** Secret value is valid JSON matching the partykit CLI config schema
**Why human:** The implementation writes the secret directly as `~/.partykit/config.json` via `printf '%s' "$PARTYKIT_TOKEN"`. If the secret is a bare token (not JSON), the CLI will fail to parse the config file. The SUMMARY states the secret stores full config JSON, but this cannot be verified from the codebase. A malformed secret would cause silent deploy failures.
**Risk:** If this is wrong, truth 3 ("live server reflects current party/index.ts") is not achieved — the workflow step would appear to run but deploy nothing.
