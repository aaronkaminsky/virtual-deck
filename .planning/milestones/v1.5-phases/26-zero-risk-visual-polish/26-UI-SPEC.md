## Color

All tokens are defined in `src/globals.css` and carried forward. No new color values are introduced in this phase.

| Role | Token | Value (HSL) | Usage |
|------|-------|-------------|-------|
| Dominant (60%) | `--background` | hsl(160 38% 16%) | Board surface, page background — dark felt green |
| Secondary (30%) | `--secondary` | hsl(160 30% 22%) | Pile card zones, spread zone backgrounds, card slot cells |
| Accent (10%) | `--primary` | hsl(38 92% 50%) | Selection ring (ring token), drop-target border highlight, selection count badge background |
| Destructive | `--destructive` | hsl(0 84% 60%) | Destructive actions only (none in this phase) |
| Muted text | `--muted-foreground` | hsl(220 9% 46%) | Zone labels, icon subtlety, drag placeholder borders |
| Border | `--border` | hsl(160 20% 20%) | Default zone/card borders |

Accent reserved for: drop-target active border (`border-primary`), selection count badge background (`bg-primary`), focus ring (`ring`). Not used for general interactive states.

Phase-specific color change: The badge at zero count (`Badge` with `pile.cards.length === 0`) is hidden — no zero-count badge renders. The badge token itself (`bg-primary`) is unchanged; conditional render eliminates it when count is 0.
