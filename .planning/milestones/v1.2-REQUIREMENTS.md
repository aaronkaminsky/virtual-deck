# Requirements Archive: Virtual Deck v1.2

**Milestone:** v1.2 Dev Infrastructure & Game Depth
**Archived:** 2026-04-29
**Original defined:** 2026-04-20
**Core Value:** Players can see the shared table and their own private hand update in real time, with no one able to see each other's face-down cards.

## v1.2 Requirements (all complete)

### Dev Infrastructure

- [x] **DEV-01**: Developer can use Playwright MCP in Claude Code sessions via a committed `.mcp.json` (project-scoped)
  - *Outcome: validated — `.mcp.json` registered `@playwright/mcp@0.0.70` via stdio; UAT confirmed MCP available in Claude Code dev sessions*
- [x] **DEV-02**: Playwright e2e test suite covers 2-player sync, deal, pass card, reset table, and hand privacy scenarios
  - *Outcome: validated — 5 scenarios + 3 more added in phases 14/15; all passing in CI and locally*
- [x] **DEV-03**: README documents local setup, architecture overview, and deploy instructions (developer-facing)
  - *Outcome: validated — 146-line README.md, VERIFICATION.md passed*
- [x] **DEV-04**: Unit test mock helpers correctly model local vs remote player; `viewFor` masking is tested for sender and recipients
  - *Outcome: validated — `tests/helpers.ts` thunk pattern; `tests/broadcastMasking.test.ts` DEV-04a–d all covered*

### Gameplay Zones

- [x] **PLAY-01**: Each connected player has a personal spread zone — all cards visible simultaneously (fanned/spread, not stacked), with a face-up/face-down toggle control like existing piles
  - *Outcome: validated — `spread-{playerToken}` pile created idempotently in `onConnect`; SpreadZone renders cascade with toggle and sortable reorder; 5 visual behaviors deferred to live session*
- [x] **PLAY-02**: A shared communal spread zone exists on the table — all cards visible simultaneously, with a face-up/face-down toggle; any player can place or move cards
  - *Outcome: validated — `play` pile converted to `region: "spread"` communal zone; rendered in BoardView spread row; no ownership restriction on MOVE_CARD*
- [x] **PLAY-03**: Player can select 1–5 cards from their hand and play them as a set into their personal zone or the communal zone in one action
  - *Outcome: validated — `PLAY_CARD_SET` server handler; click-to-select ring+lift UX; drag dispatches all selected; Playwright e2e 2-player sync test passing*

## Future Requirements (deferred beyond v1.2)

### Gameplay Zones (deferred)

- **PLAY-04**: Drag-based multi-card selection (dnd-kit multi-drag overlay) — deferred; select-then-drag is sufficient for v1.2
- **PLAY-05**: Visual set grouping / separator in spread zone — distinguishes one player's play from another's in the communal zone
- **PLAY-06**: Action log showing "Player X played N cards"

## Out of Scope

| Feature | Reason |
|---------|--------|
| Rule enforcement (legal set validation) | No game logic — sandbox; rules vary by game |
| Drag-based multi-select | Complex dnd-kit customization; select-then-button is sufficient |
| Player-facing README / how-to-play | Developer-facing only for v1.2 |
| Mobile-first layout | Drag-and-drop UX significantly worse on touch |
