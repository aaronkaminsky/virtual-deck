<objective>
Close GAP-05: when the viewport gets narrower than the board's natural minimum width,
zones overlap instead of scrolling.

Fix: remove `overflow-x-hidden` from the root board div and add `min-w-[320px]`.
With `overflow-x-hidden` removed, content that exceeds the viewport width causes the
html/body to scroll horizontally (native browser behavior). The `min-w-[320px]` sets
the floor at 320px — the smallest common phone width — so the layout is preserved down
to that size, and below it a scrollbar appears instead of elements collapsing.

Operator-selected approach: hard min-width (320px) on the board shell.

Note on overflow-x-hidden: The root div currently has `overflow-x-hidden` which clips
horizontal overflow. Child containers that manage their own horizontal overflow
(opponent hand, canvas, etc.) have their own `overflow-x-hidden` or `overflow-x-auto`
and are unaffected by this change.
</objective>
