<objective>
Implement the client-side card-fan animation that plays on every pile shuffle, triggered by the PILE_SHUFFLED server event.

Purpose: D-04 requires a card-fan (spread then collapse) animation on every pile shuffle. D-05 requires all players to see it simultaneously. This plan wires the PILE_SHUFFLED event from usePartySocket through to PileZone's visual layer.

- `PileZone.tsx` accepts `shufflingPileIds: Set<string>` prop and renders ghost card divs with CSS fan animation while `isShuffling` is true
- `src/globals.css` defines `@keyframes pile-fan-spread` for the spread-then-collapse fan
- Checkpoint: user verifies animation is visible and feels correct across two browser tabs
