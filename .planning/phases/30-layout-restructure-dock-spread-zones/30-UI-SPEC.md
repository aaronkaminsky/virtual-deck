```
  <div class="flex-shrink-0 ...">              -- opponent spreads row (NEW)
  <div class="flex-1 min-h-0 ...">             -- piles + grid row (grows)
  <div class="flex-shrink-0 px-4 py-1">        -- personal spread (existing)
  <HandZone />                                  -- local hand (existing)
</div>
```

### Opponent spreads row structure (D-01, CONTEXT.md)

```
<div class="flex items-start gap-4 px-4 flex-shrink-0">
  <div class="flex items-start gap-4 flex-1 overflow-hidden">
    {allOpponentIds.map(id => (
      <div key={id} class="flex flex-col flex-1 min-w-0 overflow-x-hidden">
        {opponentSpread && <SpreadZone interactive={false} />}
        {/* renders nothing when no spread — column still occupies flex-1 width (D-03) */}
      </div>
    ))}
  </div>
  <div class="w-7 self-start shrink-0 aria-hidden pointer-events-none" aria-hidden="true" />
  {/* spacer: matches ControlsBar trigger size="icon-sm" = size-7 = 28px */}
</div>
```

### Spacer element spec (Claude's Discretion)

- Use `aria-hidden="true"` + `pointer-events-none` (not `invisible`) — `invisible` preserves space but also renders a real focusable region in some screen readers; `aria-hidden` is more semantically correct for a pure layout spacer
- Width: `w-7` (28px) — the ControlsBar trigger is `size="icon-sm"` which resolves to `size-7` in `button.tsx`
- The ControlsBar wrapper adds `flex items-center gap-3 self-start`; since gap-3 (12px) comes between multiple siblings that don't exist in the spread row, only the trigger width (w-7) needs to be matched
- `self-start` on the spacer matches the `self-start` on the ControlsBar wrapper

### dnd-kit measurement strategy (D-07, CONTEXT.md)

- Add `measuring={{ droppable: { strategy: MeasuringStrategy.Always } }}` to `DndContext`
- Import: `import { MeasuringStrategy } from '@dnd-kit/core'`
- Location: `App.tsx` or `GameRoom.tsx` — wherever `DndContext` is currently mounted
