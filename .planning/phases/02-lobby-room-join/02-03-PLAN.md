  <action>
Start both dev servers and guide the user through 6 verification tests covering ROOM-01, ROOM-02, and UI design fidelity.

Run `npx partykit dev` and `npx vite` in background, then present the verification checklist below to the user.

**Test 1: Room creation and redirect (ROOM-01)**
1. Open http://localhost:5173/virtual-deck/ (no ?room param)
2. Verify: URL changes to `http://localhost:5173/virtual-deck/?room={8-char-code}`
3. Verify: Lobby panel appears with dark green background (#1a3a2a)
4. Verify: "Virtual Deck" title is visible at top of panel
5. Verify: Room code is displayed in amber text

**Test 2: Copy link (ROOM-01)**
1. Click the "Copy link" button
2. Verify: Button text changes to "Copied!" with a checkmark icon
3. Verify: After ~2 seconds, button text reverts to "Copy link"
4. Paste clipboard contents — verify it's a full URL with `?room=` param

**Test 3: Second player joins (ROOM-02)**
1. Copy the room URL from Test 2
2. Open a new browser tab (or incognito window for a different player)
3. Paste the room URL and navigate to it
4. Verify: Both tabs show 2 connected players in the player list
5. Verify: Each tab shows "You" for their own slot

**Test 4: Player identity persistence (ROOM-02)**
1. In one of the tabs, reload the page (F5 / Cmd+R)
2. Verify: After reload, the player list still shows the same number of players
3. Verify: The player is still identified as "You" (not a new player entry)
4. Open DevTools -> Application -> Local Storage -> verify `playerId` key exists

**Test 5: Connection status**
1. Verify: Bottom of panel shows "Connected" with an amber dot
2. Stop the PartyKit dev server (Ctrl+C in Terminal 1)
3. Verify: Status changes to "Connecting..." with a spinning icon
4. Restart PartyKit dev server
5. Verify: Status returns to "Connected" (partysocket auto-reconnects)

**Test 6: Visual design check**
1. Verify: Panel has dark green background with lighter card surface
2. Verify: Text is light gray, room code is amber
3. Verify: "Copy link" button has amber border/text
4. Verify: Inter font is loaded (check Network tab for fonts.googleapis.com)
