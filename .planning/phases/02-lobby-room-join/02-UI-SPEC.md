## Screens and States

Phase 2 has two distinct UI moments:

### Moment 1 — Root redirect (`/` with no `?room` param)

No visual UI. JavaScript generates a `nanoid` room ID immediately and calls `window.location.replace('/?room={id}')`. This is a programmatic redirect with no rendered screen. Duration: <50ms. No loading state needed.

### Moment 2 — Lobby waiting view (`/?room={id}`)

This is the only rendered screen in Phase 2.

**Layout:** Single centered column, vertically centered in viewport. Max width 480px. Background: `--background` (#1a3a2a).

**Panel** (background: `--card` #0f2419, border-radius 12px, padding 32px):
- App title (Display / 28px / semibold): "Virtual Deck"
- Room code block:
  - Label (14px / regular / muted): "Room code"
  - Code value (20px / semibold / accent `#f59e0b`): the 6–8 char nanoid value
  - Copy button (shadcn `Button` variant `outline`): "Copy link" with `Copy` lucide icon left
  - After copy: button label switches to "Copied!" with `Check` lucide icon for 2 seconds, then resets
- Separator (shadcn `Separator`)
- Player status section:
  - Heading (16px / semibold): "Players"
  - List of connected player slots (1–4): each slot shows either a filled indicator (connected) or an empty dashed placeholder (open slot)
  - Connected slot: amber dot + "You" for self, gray dot + "Player" for others
  - Open slot: dashed circle + muted text "Waiting..."
- Connection status (14px / muted, bottom of panel): "Connected" with amber dot when WS open; "Connecting..." with spinning `Loader2` icon when pending

## Interaction Contracts

| Interaction | Trigger | Response |
|-------------|---------|----------|
| Page load, no `?room` | `DOMContentLoaded` | Generate nanoid, `window.location.replace` — no render |
| Page load, `?room={id}` | `DOMContentLoaded` | Render lobby panel, open PartySocket connection |
| "Copy link" click | `onClick` | `navigator.clipboard.writeText(fullURL)`, swap button label to "Copied!" + `Check` icon, reset after 2000ms |
| WebSocket `STATE_UPDATE` | server message | Update player list in lobby panel; if `state.phase === "playing"` transition to board (Phase 3 stub) |
| WebSocket `ERROR` | server message | Show inline error below panel (see Copywriting — error state) |
| WebSocket close / reconnecting | `onclose` | Show "Reconnecting..." status, `Loader2` spin animation |
