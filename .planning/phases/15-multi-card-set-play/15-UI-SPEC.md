## Interaction Contract

This section is phase-specific and supplements the standard template.

### Selection Toggle (D-01)

- A **click** on a hand card (pointerdown + pointerup without crossing 8px distance threshold) toggles that card's selection state.
- Selection is always available — no separate "select mode" to enter.
- Clicking a selected card deselects it (same toggle mechanism).
- Clicking any non-hand-card area clears all selection. Implement via `onPointerDown` on board container with `stopPropagation` on individual card wrappers.
- Pressing Escape clears all selection. Implement via `useEffect` + `keydown` listener on document.

### Drag Sensors (D-02)

Use `useSensors` with both sensors active simultaneously:
- `PointerSensor`: `activationConstraint: { distance: 8 }` — 8px movement = drag intent (mouse)
- `TouchSensor`: `activationConstraint: { delay: 250, tolerance: 5 }` — 250ms hold = drag intent (touch)

This is the first explicit `useSensors` call in the project. The existing default sensor (no constraint) is replaced.

### Drag Behavior (D-03, D-04)

- Dragging a **selected** card: drags the entire selection as a set. On drop, dispatch `PLAY_CARD_SET` with all selected card IDs and the target zone ID.
- Dragging an **unselected** card when other cards are selected: clears selection, moves only the dragged card via normal `MOVE_CARD`. No partial set dispatch.
- After a successful set drop, selection state clears automatically (D-05).

### Drag Preview (Claude's Discretion)

Drag preview for a multi-card set: render a stacked ghost showing the top 2–3 cards offset by 4px each (bottom-right stacking). If implementation cost is high, fall back to single-card ghost using the dragged card. The stacked ghost is preferred for clarity.

### Zone Targeting (D-06, D-07)

Zone choice is determined by drop position — no intermediate dialog or button bar. Any `useDroppable` zone (SpreadZone, PileZone, HandZone) is a valid target. No zones are restricted for multi-card drops.

### Selection Count Badge (Claude's Discretion)

When 2 or more cards are selected, render a small badge near the hand label: `"N selected"`. Use `text-xs`, `bg-primary`, `text-primary-foreground`, `rounded-full px-1.5`. Position: inline after the player display name in the hand label row. If 0 or 1 card selected, badge is not rendered (not hidden — removed from DOM).

## Selection Indicator

### Visual spec for selected hand cards

| Property | Value | Class |
|----------|-------|-------|
| Ring | 2px solid primary | `ring-2 ring-primary` |
| Ring offset | Use `ring-offset-1 ring-offset-background` | Ensures ring separates from white card border |
| Upward lift | 6px translateY (negative) | `style={{ transform: 'translateY(-6px)' }}` or Tailwind `-translate-y-1.5` |
| Transition | 150ms ease | `transition-transform duration-150` |
| Rounded | Match card corner radius | `rounded-md` (existing) |

Both ring and lift apply simultaneously. The combined effect must be perceptible at the card overlap density used in HandZone (`-ml-5` / −20px overlap).

Ring offset uses `--background` token (`hsl(160 38% 16%)`) to visually separate amber ring from the white card border.

## State Architecture Note

Selection state lives as `useState<Set<string>>` in `HandZone` (preferred) or lifted to `BoardView` if cross-component coordination is required during implementation. Keep it local until a concrete need to lift emerges. Do not put selection state in zustand or on the server — this is ephemeral UI state only.

## Accessibility

- Selection ring must meet WCAG 2.1 AA contrast: amber `hsl(38 92% 50%)` on dark background — verified sufficient contrast.
- Escape key to clear selection must not conflict with any existing keyboard shortcut in the application (none currently registered).
- `aria-pressed` attribute on selected hand card wrappers: `aria-pressed={isSelected}` — conveys toggle state to screen readers.
- Drag sensors use pointer events — accessible to keyboard users only if dnd-kit keyboard sensor is added (out of scope for this phase; note as deferred).
