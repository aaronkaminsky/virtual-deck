## Position Selection UX

| Option | Description | Selected |
|--------|-------------|----------|
| Post-drop dialog | Card drops, then a small modal/popover appears: "Where? Top / Bottom / Random". Consistent with Deal popover and Reset dialog patterns. | ✓ |
| Split hover zones | Pile visually splits into top/bottom halves during drag. No random option. Complex to implement with dnd-kit. | |
| Right-click context menu | After dropping, right-click to reposition. No existing context menu pattern. Awkward UX. | |
| Pre-drop modifier key | Hold Shift for bottom, Alt for random. Invisible to new users. | |

**User's choice:** Post-drop dialog

---

## Dialog Dismissal Behavior

| Option | Description | Selected |
|--------|-------------|----------|
| Dismiss = Top | Escape/outside click defaults to Top (current behavior). No card limbo. | ✓ |
| Dismiss = cancel drop | Card snaps back. More precise but potentially surprising. | |
| No dismiss (force choice) | Modal cannot be dismissed. Guarantees intent but blocks interaction. | |

**User's choice:** Dismiss = Top (current behavior preserved)

---

## Scope: Which Piles

| Option | Description | Selected |
|--------|-------------|----------|
| All pile drops | Any card dropped on any pile triggers the position dialog. Consistent. | ✓ |
| Draw pile only | Only drops onto the 'draw' pile trigger the dialog. Narrower, matches phase name. | |
| Configurable per pile | Per-pile metadata controls whether dialog shows. Too complex. | |

**User's choice:** All pile drops

---

## What 'Random' Means

| Option | Description | Selected |
|--------|-------------|----------|
| Random = secret, no reveal | Server inserts at a random index. No one sees the position. | ✓ |
| Random = secret, reveal to sender | Server tells the sender their card's position. Extra complexity. | |
| Random = reshuffle whole pile | Dropping with Random shuffles the entire destination pile. | |

**User's choice:** Random = secret, no reveal
