# Phase 999.2 Plan 01: Put Card Back on Draw Pile Summary

**Post-drop position dialog (Top/Bottom/Random) wired end-to-end: intercepts pile drops in BoardDragLayer, sends insertPosition to server which inserts via push/unshift/splice accordingly**

## Performance

- **Duration:** ~15 min
- **Started:** 2026-04-12T00:00:00Z
- **Completed:** 2026-04-12T00:15:00Z
- **Tasks:** 3
- **Files modified:** 3

## Accomplishments
- Extended `ClientAction` MOVE_CARD with optional `insertPosition?: 'top' | 'bottom' | 'random'` — backward compatible
- Server replaces single `dest.push(card)` with position-aware branch: top=push, bottom=unshift, random=crypto splice
- BoardDragLayer intercepts pile drops: stores pendingMove state, shows dialog, dispatches MOVE_CARD with chosen position on pick
- Dismiss (Escape or click-outside) defaults to Top via `onOpenChange` — no card can enter limbo

1. **Task 1: Extend MOVE_CARD type with insertPosition** - `043f8d7` (feat)
2. **Task 2: Position-aware insertion in party/index.ts** - `2156152` (feat)
3. **Task 3: Post-drop dialog in BoardDragLayer** - `f9e0d05` (feat)

## Files Created/Modified
- `src/shared/types.ts` - Added `insertPosition?: 'top' | 'bottom' | 'random'` to MOVE_CARD ClientAction variant
- `party/index.ts` - Replaced `dest.push(card)` with position-aware logic using crypto.getRandomValues for random
- `src/components/BoardDragLayer.tsx` - Added PendingMove state, sendPendingMove helper, Dialog.Root post-drop UI

## Decisions Made
- Used `@base-ui/react/dialog` Dialog primitives directly rather than AlertDialog — AlertDialog hardcodes `disablePointerDismissal: true`, which would break click-outside dismiss (D-03 violation)
- `PendingMove` type defined at module scope for clarity; `sendPendingMove` is a plain function inside the component
- dialog renders in a Portal with a semi-transparent Backdrop; centered via fixed+translate pattern matching existing modal conventions
