## Interaction Contract

### Drag States for OpponentHand

This is the primary interaction surface for Phase 999.1.

| State | Visual | Implementation |
|-------|--------|----------------|
| Idle (no drag) | `border-transparent` 2px border | Existing |
| Drag active from own hand (not yet over zone) | `border-primary/60` 2px dashed border | New — use `useDndContext().active?.data.current?.fromZone === 'hand'` |
| Drag hovering over zone (`isOver`) | `border-primary` 2px solid border | Existing |
| Card successfully dropped | Border resets to transparent; opponent card count increments | Driven by server STATE_UPDATE |

### Drop Zone Minimum Size

When `dragIsFromHand` is true, `OpponentHand` must be visually large enough to hit. Apply `min-h-[44px]` and `min-w-[80px]` on the container div during active drag state. This prevents a zero-card opponent hand from becoming an unclickable 0px drop target.

### "Drop here" Label

When `dragIsFromHand` is true AND opponent's hand has 0 cards, render a text label "Drop to pass" inside the zone container (12px, muted-foreground color, centered). This replaces the empty card-back array. When opponent has cards, the card backs serve as sufficient affordance.

### Collision Detection

No changes to `collisionDetection` prop unless manual testing confirms `closestCenter` misfires (see RESEARCH.md Pitfall 1). If misfires are observed during verification, switch the `DndContext` to `closestCorners`. Do not add custom collision functions.

### No New Interactive Controls

This phase does not add buttons, modals, or form controls. The drag gesture is the sole new interaction. The existing `PASS_CARD` path in `ControlsBar` (pass via button) remains unchanged.

## Copywriting Contract

| Element | Copy | Notes |
|---------|------|-------|
| Drop zone affordance label (empty opponent hand) | "Drop to pass" | 12px, muted-foreground, shown only when dragIsFromHand and cardCount === 0 |
| Drop zone affordance label (non-empty opponent hand) | None — card backs are sufficient affordance | Omit label when cards are visible |
| Error: PASS_CARD fails (server returns ERROR) | No user-visible error in this phase | Server errors are not surfaced for this action in existing architecture; consistent with other actions |
| Undo confirmation | None — single-step undo via existing Undo button; no additional copy | Consistent with existing undo behavior |
| Destructive actions | None in this phase | Reset Table (existing) is unchanged |

Empty state (opponent has 0 cards, no drag active): No copy. The drop zone is not visible as a target during non-drag states. The zone renders nothing when `cardCount === 0` and `dragIsFromHand === false`.

Source: Existing copywriting patterns from ControlsBar.tsx and OpponentHand.tsx
