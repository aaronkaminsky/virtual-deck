## Interaction Contract

### Trigger Button

- Renders always in header right section, all game phases.
- `variant="outline" size="icon-sm"` — matches existing header button style.
- Icon: `<Menu className="size-4" />` from lucide-react.
- `aria-label="Open controls"` when closed; `aria-label="Close controls"` when open (driven by `aria-expanded` state).
- No label text — icon only (source: CONTEXT.md D-03).

### Popover Panel

- `side="bottom" align="end"` — anchors below the trigger, right-aligned to header edge.
- Closes on outside-click (Radix default behavior, source: CONTEXT.md D-05).
- Panel width: `w-56` (224px) — wide enough for inline card count stepper + Deal button.
- Panel padding: `p-4` (16px) on all sides.

### Panel Layout (top → bottom)

```
[ Copy link ]                  ← full-width Button, variant="outline", size="sm"
────────────── Separator ──────
  Cards per player             ← text-sm font-semibold label
  [−][ 5 ][+]       [Deal]    ← stepper + Button variant="default" size="sm"
────────────── Separator ──────
  [Undo]    [Reset table]      ← two buttons, side by side, variant="outline"/"destructive" size="sm"
```

Source: CONTEXT.md D-06 and Specifics section (confirmed by user).

### Auto-Close Behavior

Panel auto-closes after any completed action (source: CONTEXT.md D-04):
- Copy link: closes after "Copied!" feedback (~1.5s setTimeout — replicates existing `BoardView.tsx` pattern).
- Deal: closes immediately after `sendAction({ type: 'DEAL_CARDS' })`.
- Undo: closes immediately after `sendAction({ type: 'UNDO_MOVE' })`.
- Reset (confirmed): closes immediately after `sendAction({ type: 'RESET_TABLE' })`.

### Enabled / Disabled States

| Control | Enabled condition |
|---------|------------------|
| Deal | `gameState.phase === 'setup' \|\| gameState.phase === 'lobby'` |
| Undo | `gameState.canUndo === true` |
| Reset table | `gameState.phase === 'playing'` |
| Copy link | Always enabled |

Controls are rendered in all phases but disabled via `disabled` prop when not applicable (source: CONTEXT.md D-06). No conditional rendering by phase.

### Two-Step Reset Confirmation

Inline within the Popover panel — no AlertDialog (source: CONTEXT.md D-07).

State: `confirmReset: boolean` (via `useState`, default `false`).

- **Step 1** — Reset row shows: `<Button variant="destructive" size="sm"><RotateCcw /> Reset table</Button>`
- **Step 2** — After click, same row morphs to: `<span className="text-sm text-muted-foreground">Are you sure?</span> <Button variant="outline" size="sm" onClick={cancel}>Cancel</Button> <Button variant="destructive" size="sm" onClick={confirm}>Reset table</Button>`
- Clicking Cancel returns to Step 1 (`confirmReset = false`). Clicking Reset table fires `sendAction` and closes panel.

### Copy Link

- Moves from `BoardView.tsx` header into ControlsBar Popover panel as first item (source: CONTEXT.md D-09).
- `roomId` prop added to `ControlsBarProps`.
- Copies `${window.location.origin}${BASE_URL}?room=${roomId}` to clipboard.
- Button shows `<Copy /> Copy link` → `<Check /> Copied!` for 1.5s, then panel closes (source: CONTEXT.md D-10).

## Copywriting Contract

| Element | Copy | Source |
|---------|------|--------|
| Trigger button aria-label (closed) | "Open controls" | Default |
| Trigger button aria-label (open) | "Close controls" | Default |
| Copy link button | "Copy link" | Existing (BoardView.tsx) |
| Copy link success feedback | "Copied!" | Existing (BoardView.tsx) |
| Deal section label | "Cards per player" | Existing (ControlsBar.tsx) |
| Deal button | "Deal" | Existing (ControlsBar.tsx) |
| Undo button | "Undo" | Existing (ControlsBar.tsx) |
| Reset button (step 1) | "Reset table" | Existing (ControlsBar.tsx) |

No error state needed — clipboard failure is silent per existing `BoardView.tsx` pattern (`.catch(() => {})`).
