## Spacing Scale

Declared values (multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline badge padding |
| sm | 8px | Card overlap calculation basis, compact buttons |
| md | 16px | Zone horizontal padding (px-4), header padding |
| lg | 24px | Section gaps between zones |
| xl | 32px | Layout column gaps |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing |

Exceptions:
- Touch target minimum: 44px height (already applied on `OpponentHand` drop target via `min-h-[44px]` during drag-active state — no change needed)
- Card overlap (hand): `-ml-5` (20px) at desktop; `-ml-3` (12px) at phone (source: CONTEXT.md D-04, codebase HandZone.tsx line 39)
- Card overlap (spread): `-ml-5` (20px) at desktop; `-ml-3` (12px) at phone (proportional to card size reduction)

## Color

All color tokens are pre-established in `src/globals.css`. This phase makes no color changes — only layout and sizing changes.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `--background`: hsl(160 38% 16%) | Board surface — dark green felt |
| Secondary (30%) | `--card`: hsl(160 38% 10%) / `--secondary`: hsl(160 30% 22%) | Header bar, HandZone strip, zone backgrounds, spread zone slots |
| Accent (10%) | `--primary`: hsl(38 92% 50%) amber/gold | Reserved for: selected card ring, active drop target border, connection banner, badge background, "selected" count badge |
| Destructive | `--destructive`: hsl(0 84% 60%) | Destructive confirmation actions in ControlsBar (reset table) only |

Accent reserved for (explicit list):
- Selected card ring (`ring-1 ring-primary/30`) in HandZone
- Active drop target border (`border-primary`) on HandZone, PileZone, SpreadZone, OpponentHand
- Drag-hover state border on drop zones
- ControlsBar trigger glow/ring token (`--ring`)
- "N selected" count badge on HandZone

Accent is NOT used for: general text, zone labels, card backs, inactive borders, or scrollbars.

Source: `src/globals.css` :root — detected from codebase. No changes in this phase.

## Responsive Breakpoints

This is the core contract for Phase 19. All breakpoints use Tailwind's default `sm:` threshold.

| Breakpoint | Viewport | Behavior |
|------------|----------|----------|
| Default (phone) | < 640px | Phone sizing — all overrides apply |
| `sm:` | >= 640px | Desktop sizing — existing behavior preserved |

### Card Sizes

| Context | Phone (default) | Desktop (sm:) | Source |
|---------|-----------------|---------------|--------|
| CardFace (image path) | `w-[42px] h-[59px]` | `sm:w-[63px] sm:h-[88px]` | CONTEXT.md D-02 / D-03 |
| CardFace (fallback div) | `w-[42px] h-[59px]` | `sm:w-[63px] sm:h-[88px]` | CONTEXT.md D-02 |
| CardBack (image path) | `w-[42px] h-[59px]` | `sm:w-[63px] sm:h-[88px]` | CONTEXT.md D-02 |
| CardBack (fallback div) | `w-[42px] h-[59px]` | `sm:w-[63px] sm:h-[88px]` | CONTEXT.md D-02 |
| OpponentHand card | `w-[42px] h-[59px]` | already correct, no change | CONTEXT.md D-03 |

### Zone Container Sizes

| Zone | Phone (default) | Desktop (sm:) | Source |
|------|-----------------|---------------|--------|
| HandZone card wrapper | `w-[42px] h-[59px]` | `sm:w-[63px] sm:h-[88px]` | proportional to card |
| HandZone container height | `h-[100px]` | `sm:h-[128px]` | proportional: 59+~41px margin |
| PileZone card slot | `w-[56px] h-[79px]` | `sm:w-[80px] sm:h-[112px]` | proportional 42/63 ratio ≈ 0.667 |
| SpreadZone card slot | `min-w-[56px] h-[79px]` | `sm:min-w-[80px] sm:h-[112px]` | proportional |

Note: PileZone and SpreadZone container dimensions are 80×112px at desktop (existing). Phone equivalent is 80×0.667 ≈ 56px wide, 112×0.667 ≈ 79px tall. Round to nearest 4px.

### Overlap (negative margin)

| Context | Phone (default) | Desktop (sm:) | Source |
|---------|-----------------|---------------|--------|
| HandZone `-ml-*` | `-ml-3` (12px) | `sm:-ml-5` (20px) | CONTEXT.md specifics — 29% vs 32% overlap |
| SpreadZone `-ml-*` | `-ml-3` (12px) | `sm:-ml-5` (20px) | proportional match |

### Overflow / Scroll

| Element | Phone (default) | Desktop (sm:) | Source |
|---------|-----------------|---------------|--------|
| BoardView root div | `overflow-x-hidden overflow-y-auto` | `sm:overflow-hidden` | CONTEXT.md D-05 / D-06 |
| Opponents strip | `overflow-x-auto` (unchanged) | `overflow-x-auto` (unchanged) | CONTEXT.md D-01 |
| HandZone inner | `overflow-x-auto` (unchanged) | `overflow-x-auto` (unchanged) | existing, no change |

Note: `h-screen` on BoardView root stays at both breakpoints. Only overflow behavior changes.
