## Component Inventory

New and modified components for this phase.

### Toggle Button (new, inside HandZone header row)

```
Variant:    ghost
Size:       h-7 w-7 p-0  (28×28px — matches PileZone.tsx:77-94)
Icon:       Eye (w-4 h-4) when hand is HIDDEN (normal state)
            EyeOff (w-4 h-4) when hand is REVEALED
title:      "Show hand to opponents" | "Hide hand from opponents"
aria-label: "Show hand" | "Hide hand"
Position:   Inline in HandZone header row, after the selection badge, before any future controls
```

Icon semantics: `Eye` = "click to show" (hand is currently hidden); `EyeOff` = "click to hide" (hand is currently revealed). This matches the affordance pattern in PileZone where `Eye` means "face-up landing" (the active/visible state). Apply the same logic here: the icon describes what WILL happen, not the current state — but given CONTEXT.md D-02 says "EyeOff when revealed," implement as: show `EyeOff` icon when `isRevealed === true` (indicating you can click to hide).

### HandZone Container (modified)

Add conditional ring when `isRevealed === true`:

```
Base classes (unchanged): h-[100px] sm:h-[128px] flex items-center px-4 overflow-x-auto bg-card
Drag-over (unchanged):    border-t-2 border-primary
Revealed state (NEW):     ring-1 ring-primary/50 ring-inset
```

Full `cn()` expression:
```tsx
cn(
  'h-[100px] sm:h-[128px] flex items-center px-4 overflow-x-auto bg-card',
  isOver ? 'border-t-2 border-primary' : '',
  isRevealed ? 'ring-1 ring-primary/50 ring-inset' : ''
)
```

Source for ring value: CONTEXT.md D-05 + RESEARCH.md Pattern 4. Ring opacity `/50` is stronger than selection ring `/30` — intentionally more prominent since reveal state is a persistent condition, not a momentary hover.

### OpponentHand — Revealed Path (new branch)

When `revealedCards` is non-empty, replace the CardBack row with a CardFace row:

```
Layout:     flex items-center (same as back row)
Card size:  w-[42px] h-[59px]  (same as back row — no responsive upsize for opponent cards)
Overlap:    -ml-3 on i > 0  (same as back row)
Cap:        NONE — all revealed cards shown (D-03: no card count cap)
Scroll:     overflow-x-auto on the parent flex row
```

The outer `OpponentHand` container dimensions and border behavior are unchanged. Only the inner card row content changes.

## Interaction Contract

### Toggle: Show Hand

Trigger: Player clicks the Eye icon button in their own HandZone header.

Flow:
1. Click → `onToggleReveal()` → dispatch `SET_HAND_REVEALED { revealed: true }` via `sendAction`
2. Server updates `player.handRevealed = true`, broadcasts state
3. Local player: `isRevealed` prop becomes `true` → EyeOff icon shown + `ring-1 ring-primary/50 ring-inset` applied to HandZone container
4. All opponents: `revealedCards` prop on their `OpponentHand` becomes populated → CardFace row rendered

No confirmation required — showing your hand is reversible.

### Toggle: Hide Hand

Trigger: Player clicks the EyeOff icon button in their own HandZone header.

Flow:
1. Click → `onToggleReveal()` → dispatch `SET_HAND_REVEALED { revealed: false }` via `sendAction`
2. Server updates `player.handRevealed = false`, broadcasts state
3. Local player: Eye icon shown, ring removed
4. All opponents: `revealedCards` prop becomes undefined/empty → CardBack row rendered (count from `opponentHandCounts`)

No confirmation required — hiding is safe.

### Reset Table

No UI changes for this phase. `RESET_TABLE` is an existing action. The server clears all `handRevealed` flags (D-07). The result is that all HandZones return to hidden state, which flows automatically through the existing state broadcast.

### Reconnect

No UI state to restore on reconnect. `handRevealed` is persisted server-side in `GameState`. When a client reconnects and receives the first `STATE_UPDATE`, `myHandRevealed` and `opponentRevealedHands` are populated from persisted state — the UI renders correctly without any client-side action.

| State | HandZone (local player) | OpponentHand (other players) |

| Drag-over (hidden) | `border-t-2 border-primary` (existing) | `border-2 border-primary` (existing) |
| Drag-over (revealed) | `border-t-2 border-primary` + `ring-1 ring-primary/50 ring-inset` | `border-2 border-primary` |
