## Trust Boundaries

| Boundary | Description | Data Crossing |
|----------|-------------|---------------|
| client UI state | pendingMove is local ephemeral client state; no server action is sent unless the user explicitly clicks a position button | None — purely local React state, no network I/O |

---

## Threat Register

| Threat ID | Category | Component | Disposition | Mitigation | Status |
|-----------|----------|-----------|-------------|------------|--------|
| T-999.11-01 | Tampering | pendingMove client state | accept | pendingMove is ephemeral UI state; clearing it without sending has no server-side consequence. The server never received the MOVE_CARD for the pending move, so no revert is needed and no inconsistency can arise. | closed |
| T-999.11-02 | Denial of Service | Dialog dismiss race | accept | If the dialog closes and a card move is dropped, the card simply stays at its origin — the server state is always valid. No exploit surface. | closed |

*Status: open · closed*
*Disposition: mitigate (implementation required) · accept (documented risk) · transfer (third-party)*
