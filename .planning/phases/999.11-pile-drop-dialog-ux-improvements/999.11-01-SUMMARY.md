# Phase 999.11 Plan 01: Pile Drop Dialog UX Improvements Summary

Pile insert-position dialog now cancels on dismiss (Escape/click-outside) and auto-focuses the Top button so Enter confirms top placement without a mouse click.

## Tasks Completed

| Task | Description | Commit |
|------|-------------|--------|
| 1 | Rewrite pile dialog dismiss logic and keyboard UX | 7348217 |

## What Was Built

Updated `BoardDragLayer.tsx` pile insert-position dialog:

- `onOpenChange` handler replaced: any dismissal now calls `setPendingMove(null)` (cancel) instead of `sendPendingMove('top')` (confirm). Card stays at server origin on Escape or click-outside.
- `Dialog.Popup` gains `initialFocus={topButtonRef}` — focus moves to the Top button when dialog opens, so pressing Enter confirms Top.
- Three raw `<button>` elements replaced with `<Button>` components: Top uses `variant="default"` (filled/primary), Bottom and Random use `variant="outline"`.
- `button.tsx` updated to use `React.forwardRef` so `ref={topButtonRef}` is type-safe.

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 2 - Missing Critical Functionality] Button component lacked forwardRef**
- **Found during:** Task 1
- **Issue:** The plan stated Button was "forward-ref compatible via ButtonPrimitive", but the wrapper function was a plain function without `React.forwardRef`. Passing `ref={topButtonRef}` would fail TypeScript type-checking in React 18.
- **Fix:** Wrapped `Button` in `React.forwardRef`, passing the ref through to `ButtonPrimitive`. Added `Button.displayName = "Button"`.
- **Files modified:** `src/components/ui/button.tsx`
- **Commit:** 7348217
