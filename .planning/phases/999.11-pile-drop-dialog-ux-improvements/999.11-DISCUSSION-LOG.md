## Click-outside behavior

| Option | Description | Selected |
|--------|-------------|----------|
| Cancel (same as Escape) | Any dismissal = cancel. Consistent mental model: the only way to commit is to pick a position. | ✓ |
| Confirm Top (keep current) | Click-outside = "I don't care, just do Top." Forgiving; Escape is the explicit cancel path. | |

**User's choice:** Cancel (same as Escape)
**Notes:** Consistent model — any dismissal = cancel.

---

## Top button visual

| Option | Description | Selected |
|--------|-------------|----------|
| Primary button style | Top gets filled/solid primary style; Bottom and Random stay bordered. Clear visual hierarchy. | ✓ |
| Add (Enter) hint label | Top button label becomes "Top (Enter)". All three buttons keep equal styling. | |
| No visual change | Wire Enter silently, no UI change. | |

**User's choice:** Primary button style
**Notes:** Visual hierarchy signals Top as default.

---

## Enter key implementation

| Option | Description | Selected |
|--------|-------------|----------|
| Auto-focus Top button on open | Dialog opens with Top focused. Enter clicks it naturally. Tab cycles Bottom → Random → Top. | ✓ |
| Explicit keydown handler | onKeyDown on Dialog.Popup intercepts Enter regardless of focus. | |

**User's choice:** Auto-focus Top button on open
**Notes:** Clean, accessible, no bespoke key handler.
