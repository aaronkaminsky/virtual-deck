## Interaction States

All states are defined prescriptively. Executor must implement exactly these, not approximations.

### Canvas Card: Unselected (at rest)

```
boxShadow: coversAnother ? STACK_SHADOW : undefined
borderRadius: coversAnother ? 6px : undefined
opacity: 1
cursor: grab
```

`STACK_SHADOW = '2px 2px 0 0 #fff, 2px 2px 0 1px #d1d5db'` (from `canvas-utils.ts`)

### Canvas Card: Selected (at rest, not dragging)

```
boxShadow: '0 0 0 2px #60a5fa, 0 0 0 4px rgba(96,165,250,0.3)' +
           (coversAnother ? ', 2px 2px 0 0 #fff, 2px 2px 0 1px #d1d5db' : '')
borderRadius: 6px
opacity: 1
cursor: grab
```

The selection ring uses `boxShadow` (not CSS `ring` classes) because canvas cards are absolutely positioned тАФ Tailwind ring utilities conflict with the stack shadow on absolutely positioned elements.

### Canvas Card: Source card during active group drag (invisible тАФ position placeholder)

```
opacity: 0
```

All cards in the active drag group (handle + passengers) set `opacity: 0` on their source element. This matches the existing single-card drag behavior in `CanvasDraggableCard`.

### DragOverlay тАФ drag handle card

```
opacity: 0.5
transform: scale(1.05)
cursor: grabbing
boxShadow: dragOverlapsAnother ? STACK_SHADOW : undefined
borderRadius: dragOverlapsAnother ? 6px : undefined
dropAnimation: null
```

Source: Phase 32 decision D-13 / Spike 004 DragOverlay pattern.

### Passenger ghost cards (non-handle selected cards during drag)

```
position: absolute
zIndex: 998
opacity: 0.5
pointerEvents: none
left: card.x + delta.x
top: card.y + delta.y
boxShadow: dragOverlapsAnother ? STACK_SHADOW : undefined
borderRadius: dragOverlapsAnother ? 6px : undefined
```

Passenger ghosts render inside `CanvasZone` at `zIndex: 998`. The drag handle card in `DragOverlay` renders above them at the browser's overlay z-context. All passengers match the handle opacity (0.5) for a uniform translucent group appearance.

### Bounds violation тАФ cancelled group drop

No visual feedback. Silent snap-back. All cards return to pre-drag positions. This is consistent with existing `onDragCancel` behavior (Phase 32 decision D-15).

Do not add:
- Toast or snackbar message
- Error highlight or red border
- Shake animation

### Canvas zone тАФ drag-active drop indicator

```
ring-1 ring-primary/30  (existing, no change)
```

Already implemented in `CanvasZone`. No change for group drop.

### Selection count badge

When 2 or more canvas cards are selected, display a count badge to the right of the canvas zone label (matching the hand zone pattern in `HandZone`).

```
<span className="ml-2 text-xs bg-primary text-primary-foreground rounded-full px-1.5">
  {count} selected
</span>
```

Show only when `selectedIds.size >= 2` and `selectionSource.zone === 'canvas'`.

| Requirement | Implementation |
|-------------|----------------|

| Deselect on canvas click | `onClick` on canvas background, not `onPointerDown` тАФ avoids dnd-kit listener conflict |
