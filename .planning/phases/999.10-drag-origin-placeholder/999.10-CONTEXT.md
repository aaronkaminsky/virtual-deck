# Phase 999.10: Drag Origin Placeholder — Context

**Gathered:** 2026-04-14
**Status:** Ready for planning

<domain>
## Phase Boundary

When a player drags a card, a dashed card-shaped outline appears in the slot where the card originated, so they can see where it came from while it's in flight. The placeholder disappears once the card lands (successful drop) or once the snap-back animation completes (cancelled drag).

**In scope:**
- Dashed outline placeholder at the origin slot for pile-to-anywhere drags
- Dashed outline placeholder at the origin slot for hand-to-pile or hand-to-opponent-hand drags (cross-zone only)
- Placeholder persists during the snap-back animation on drag cancel, then disappears when card lands

**Out of scope:**
- Hand-to-hand reordering (within the same hand) — dnd-kit sortable's shift behavior is sufficient; no separate placeholder
- Any changes to the drag overlay (CardOverlay) or snap-back animation timing

</domain>

<decisions>
## Implementation Decisions

### Appearance
- **D-01:** Placeholder style: dashed card-shaped border, empty inside. Matches the existing empty pile border style (`border-dashed` already used in PileZone for empty piles) — reuse the same visual language.

### Zone Scope
- **D-02:** Placeholder appears in **both** hand and pile origin slots.
  - Pile: when the top card is dragged off, the pile zone shows the dashed outline (instead of going invisible at opacity 0).
  - Hand: when a card is dragged cross-zone (to a pile or opponent hand), the hand slot shows a dashed outline in place of the card.
- **D-03:** Hand-to-hand reorder (drag within own hand) — **no placeholder**. dnd-kit sortable's existing card-shift animation is the feedback. Adding a dashed origin slot alongside the sortable shift would create conflicting visual signals.

### Cancel / Snap-Back Behavior
- **D-04:** On drag cancel, the placeholder **persists** until the snap-back animation completes. The card animates back to the slot occupied by the placeholder, then the placeholder disappears when the card renders at full opacity. This prevents a visual "jump" where the slot looks empty during the return animation.

### Implementation Approach
- **D-05:** Implement in `DraggableCard` (pile cards) and `SortableHandCard` (hand cards) by replacing `opacity: isDragging ? 0 : 1` with a rendered placeholder element when `isDragging` is true. The draggable wrapper remains in the DOM (dnd-kit requires this for position tracking); render the dashed outline in place of the card face/back.
- **D-06:** The placeholder is purely client-local visual state — no server action, no game state change.

### Claude's Discretion
- Exact CSS for the dashed outline (border width, color token — should match `border-border` or `border-muted` from the dark felt theme)
- Whether the placeholder uses a separate component or inline conditional rendering
- Timing of placeholder disappearance relative to `defaultDropAnimation.duration` on cancel

</decisions>

<code_context>
## Existing Code Insights

### Reusable Assets
- `border-dashed` + `border-border` already in `PileZone` for empty pile state — placeholder should match this exactly for visual consistency
- `defaultDropAnimation.duration` (imported in BoardDragLayer) is the snap-back timing reference

### Established Patterns
- `isDragging` boolean is already available in both `useDraggable` (DraggableCard) and `useSortable` (SortableHandCard) — no new state needed
- The draggable `div` wrapper must stay in the DOM during drag for dnd-kit position tracking; the placeholder replaces the card face/back content, not the wrapper itself

### Integration Points
- `SortableHandCard` handles both hand-to-hand reorder AND cross-zone hand drags. The `isDragging` flag is true in both cases. The decision (D-03) to skip placeholder for hand-to-hand reorder means the placeholder should apply to all `isDragging` states in `SortableHandCard` — the sortable shift for reorder is provided by dnd-kit and won't conflict visually since the origin card is what shifts, not a separate placeholder.

</code_context>

<specifics>
## Specific Ideas

- The dashed outline should feel like a "ghost slot" — the same size as a card (80×112px in PileZone), same rounded corners, just an empty bordered box
- Visually consistent with how empty piles already look — players already know what that means

</specifics>
