# Phase 999.10 Plan 02: Missed Hand-Drop Snap-Back Summary

**One-liner:** Split `isHandToHand` into `isHandReorder` / `isHandMissed` so releasing a card over the empty hand strip triggers snap-back animation instead of instant vanish.

## What Was Built

`handleDragEnd` in `BoardDragLayer.tsx` now distinguishes two hand-to-hand cases:

- `isHandReorder` — drop lands on a specific sortable card id (real reorder occurred). Overlay clears immediately, same as before.
- `isHandMissed` — drop lands on the `'hand'` droppable id itself (user released over empty hand strip, no card underneath). Treated as a failed drop: `dropSuccessRef.current` stays false, so `DragOverlay` uses `defaultDropAnimation` (snap-back) rather than `null` (instant vanish). `setActiveCard(null)` is delayed until after the animation completes.

The dashed placeholder rendered in `SortableHandCard` (from plan 01) remains visible at the card's origin slot throughout the snap-back because `isDragging` is cleared via `setDragging(false)` only after the animation timeout.
