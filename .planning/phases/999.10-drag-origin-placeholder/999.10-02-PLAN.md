  <action>
In `handleDragEnd`, replace the single `isHandToHand` variable with two narrower variables:

- `isHandReorder` — hand-to-hand drop where `event.over.id` is a specific card ID (a real sort happened). The overlay clears immediately, same as before.
- `isHandMissed` — hand-to-hand drop where `event.over.id === 'hand'` (the droppable zone itself, not a card). No sort happened. Treat as a failed drop: use the delayed `setActiveCard(null)` path so snap-back animation plays and the placeholder stays visible.

**BEFORE (lines 56–95):**

```tsx
const overData = event.over?.data.current as { toZone: string; toId: string } | undefined;
const isHandToHand = dragDataRef.current?.fromZone === 'hand' && overData?.toZone === 'hand';
const isPassCard = !!(overData?.toZone === 'opponent-hand');
const isSuccess = !!(event.over && dragDataRef.current && overData?.toZone && !isHandToHand && !isPassCard);
dropSuccessRef.current = isSuccess || isHandToHand || isPassCard;
setDragging(false);

if (isPassCard) {
  setActiveCard(null);
  // ...pass card action...
} else if (isSuccess) {
  setActiveCard(null);
  // ...move card action...
} else if (isHandToHand) {
  // Reorder handled by useDndMonitor in HandZone — just clear the overlay
  setActiveCard(null);
}
// Failed drop: keep activeCard set so overlay has content during snap-back animation.
else {
  setTimeout(() => setActiveCard(null), defaultDropAnimation.duration + 50);
}
```

**AFTER:**

```tsx
const overData = event.over?.data.current as { toZone: string; toId: string } | undefined;
const isHandReorder = dragDataRef.current?.fromZone === 'hand' && overData?.toZone === 'hand' && event.over?.id !== 'hand';
const isHandMissed = dragDataRef.current?.fromZone === 'hand' && event.over?.id === 'hand';
const isPassCard = !!(overData?.toZone === 'opponent-hand');
const isSuccess = !!(event.over && dragDataRef.current && overData?.toZone && !isHandReorder && !isHandMissed && !isPassCard);
dropSuccessRef.current = isSuccess || isHandReorder || isPassCard;
setDragging(false);

if (isPassCard) {
  setActiveCard(null);
  // ...pass card action (unchanged)...
} else if (isSuccess) {
  setActiveCard(null);
  // ...move card action (unchanged)...
} else if (isHandReorder) {
  // Reorder handled by useDndMonitor in HandZone — just clear the overlay
  setActiveCard(null);
}
// Failed drop (including isHandMissed): keep activeCard set so overlay has content
// during snap-back animation. Clear after animation completes.
else {
  setTimeout(() => setActiveCard(null), defaultDropAnimation.duration + 50);
}
```

Key changes:
1. `isHandToHand` → split into `isHandReorder` (over a card id) and `isHandMissed` (over the 'hand' droppable id).
2. `dropSuccessRef.current` no longer includes `isHandMissed`, so `DragOverlay` uses `defaultDropAnimation` (snap-back) instead of `null` (instant vanish).
3. `isHandMissed` falls through to the `else` branch — delayed `setActiveCard(null)` — so the placeholder stays visible during the animation.
4. The `isSuccess` guard is updated to exclude both `isHandReorder` and `isHandMissed`.

All other branches (`isPassCard`, `isSuccess`, `isHandReorder`, the `else` fallthrough) are otherwise identical to the original.
