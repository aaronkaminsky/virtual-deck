**Note on implementation deviation:** Plan 01 specified driving the placeholder from `useDraggable().isDragging` / `useSortable().isDragging` directly inside `DraggableCard` and `SortableHandCard`. The actual implementation uses `draggingCardId` prop threaded from `BoardDragLayer.activeCard`. This was intentional: dnd-kit's context `active` becomes null when the pointer crosses droppable zone boundaries, causing the `isDragging` hook value to flicker mid-drag. The prop-threading approach bypasses this and correctly persists the placeholder for the full drag lifetime. `DraggableCard.tsx` still has `opacity: isDragging ? 0 : 1` (hides the card during drag); the placeholder is rendered as a sibling overlay in `PileZone`. The functional outcome — visible dashed outline at origin for the full drag duration — is equivalent to the planned approach.
