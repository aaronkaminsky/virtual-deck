## Visual States

### Stack Shadow Indicator (OVERLAP-03)

Appears on the **covering (top) card** ‚Äî the card whose position overlaps >50% of a lower card's area.

| Property | Value | Source |
|----------|-------|--------|
| `boxShadow` | `'2px 2px 0 0 #fff, 2px 2px 0 1px #d1d5db'` | Spike002 line 47 (`STACK_SHADOW`) |
| `borderRadius` | `6px` | Spike002 line 77 ‚Äî prevents shadow from bleeding outside card corners |
| Threshold formula | `overlapW * overlapH > CARD_W * CARD_H * 0.5` | CONTEXT.md D-03 |
| Shadow subject | Covering (top) card, not the covered card | CONTEXT.md D-02 |

The white inner layer (`2px 2px 0 0 #fff`) creates a "floating paper" highlight; the gray outer border (`2px 2px 0 1px #d1d5db`) provides the depth separation against the dark felt.

### Drag Opacity (OVERLAP-02)

Already implemented in Phase 32. No changes to this state in Phase 33.

| Property | Value | Location |
|----------|-------|----------|
| `opacity` | `0.5` | `BoardDragLayer.tsx` DragOverlay div, line ~421 |
| `transform` | `'scale(1.05)'` | Same div |

### Drag Overlay Shadow

When the dragged card's current position (stored x/y + live delta) covers >50% of any non-dragged canvas card, the DragOverlay ghost also shows the stack shadow.

| Property | Value |
|----------|-------|
| `boxShadow` | `STACK_SHADOW` (same constant as static case) |
| `borderRadius` | `6px` |
| Condition | `dragCoversSomeCard === true` |
| State management | `dragCoversSomeCard: boolean` in React state in `BoardDragLayer`; `dragDeltaRef = useRef({x:0, y:0})` tracks delta without re-renders; setState called only on boolean flip |

Source: CONTEXT.md D-05, D-06; Spike002 lines 100‚Äì104.

### Source Card During Drag (OVERLAP-01 / existing)

The source card at its resting position renders `opacity: 0` while `isDragging === true` (already implemented, D-13). CSS z-index at `canvasCard.z` routes pointer events naturally ‚Äî no `pointer-events: none` manipulation required (CONTEXT.md D-01).

## Interaction Contract

### Hit-Testing (OVERLAP-01)

No new UI control or gesture. The interaction contract is:

- At any (x, y) point where two canvas cards overlap, only the card with the higher `zIndex` responds to pointer events.
- This is delivered by the browser's natural pointer-event routing through the rendered `zIndex` stack.
- No custom hit-testing code is needed; verify the existing behavior from Phase 32 and ship.

### Shadow Activation Threshold

- Threshold: overlap area > 50% of `CARD_W √ó CARD_H` (63 √ó 88 = 5544 px¬≤; threshold = 2772 px¬≤).
- Threshold applies independently to each (covering, covered) card pair.
- A card can both cover a lower card and be covered by a higher card simultaneously.

### Drag-Time Shadow Performance Contract

- `dragDeltaRef = useRef({x: 0, y: 0})` absorbs every `onDragMove` event without causing a re-render.
- `coversMajority` is computed in `onDragMove`; if the result changes the boolean, `setDragCoversSomeCard(newValue)` fires one state update.
- The re-render rate is bounded by boolean transitions, not by pointer move frequency.

Source: CONTEXT.md D-06; CLAUDE.md `isDraggingRef = useRef(false)` pattern.
