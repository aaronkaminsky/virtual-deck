## Typography

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Heading | 28px (1.75rem) | 600 (semibold) | 1.2 | Lobby panel "Virtual Deck" title — matches existing `text-[1.75rem] font-semibold leading-[1.2]` |
| Subheading | 20px (1.25rem) | 600 (semibold) | 1.2 | Room code display — matches existing `text-[1.25rem] font-semibold` |
| Body | 16px (1rem) | 400 (regular) | 1.5 | Lobby section headers, player list items |
| Label | 14px (0.875rem) | 400 (regular) | 1.5 | Muted labels ("Room code", "Connecting..."), hand zone name labels, presence text |

Name labels near hand zones use the Label tier (14px, weight 400, `text-muted-foreground`). This keeps them lightweight and avoids competing with the cards themselves.

Sources: CONTEXT.md discretion note ("lightweight, not compete with cards"); existing LobbyPanel.tsx typography patterns.

## Component Inventory

Components touched in this phase and their design specification:

### 1. LobbyPanel — Name Input Section (new)

Added above the existing Players list section, below the room code section.

```
[ Your name __________________ ]    ← shadcn Input, maxLength=20, placeholder="Your name"
[ Join Game                    ]    ← shadcn Button, disabled until trim().length > 0
```

- Input: `<Input maxLength={20} placeholder="Your name" />` — no border override, uses `--input` background
- Button: `<Button disabled={name.trim().length === 0}>Join Game</Button>` — primary variant, `min-h-[44px]`
- Button disabled state: `opacity-50 cursor-not-allowed` (shadcn built-in disabled styling)
- No inline error state — button stays disabled until valid (D-02)
- Name pre-fills from localStorage on mount (D-07)
- Input label: `<p className="text-sm text-muted-foreground mb-1">Your name</p>` — matches existing "Room code" label pattern

### 2. Hand Zone Name Label (new — HandZone.tsx and OpponentHand.tsx)

Seat label positioned above each hand zone strip. Same treatment for self and opponents (D-06).

```
● Aaron                         ← presence dot + name, single row
[card][card][card][card]        ← existing hand zone
```

Layout:
```tsx
<div className="flex items-center gap-1.5 px-1 mb-1">
  <span className={cn(
    'rounded-full inline-block w-2 h-2',
    player.connected ? 'bg-green-500' : 'bg-gray-500'
  )} />
  <span className="text-xs text-muted-foreground">
    {player.displayName || 'Player'}
  </span>
</div>
```

- Dot size: 8px × 8px (`w-2 h-2`) — matches existing `PlayerPresence.tsx` non-self dot size
- Gap between dot and name: 6px (`gap-1.5`) — matches existing `PlayerPresence.tsx` row gap
- Name text: 12px (`text-xs`), `text-muted-foreground` — sits below the label tier to feel like a seat tag
- No animation on disconnect/reconnect — state updates via React re-render when `player.connected` changes; no transition needed given the functional nature of the indicator

Sources: CONTEXT.md D-04, D-06; RESEARCH.md Pattern 3; existing `PlayerPresence.tsx` class values.

### 3. PlayerPresence — Removed (D-05)

The `PlayerPresence` component in the header bar is removed. Board-level seat labels (above) replace its function. No replacement header element is added.
