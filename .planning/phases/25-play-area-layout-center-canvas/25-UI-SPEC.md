## Spacing Scale

Declared values (multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline button padding (`p-0` icon buttons stay at h-7 w-7) |
| sm | 8px | Gap between label and controls in pile header row (`gap-1` = 4px, `gap-2` = 8px) |
| md | 16px | Horizontal board padding (`px-4`) |
| lg | 24px | Default inter-zone row padding (`py-2` → reduced to `py-1` per POLISH-03) |
| xl | 32px | Major layout gaps between board bands |
| 2xl | 48px | — |
| 3xl | 64px | — |

Exceptions:
- Card zone heights are not on the 8-point scale by design (card aspect ratio dictates them). Declared explicitly:
  - Mobile pile/spread card height: 64px (reduced from 79px per POLISH-04, ~19% reduction)
  - Desktop pile/spread card height: 90px (reduced from 112px per POLISH-04, ~20% reduction)
  - Mobile pile width: 56px (unchanged)
  - Desktop pile width: 80px (unchanged)
- Slim drop-target strip height (ZONE-01): 40px mobile / 56px desktop (approximately half card height, per CONTEXT.md D-07)
- Icon touch targets: 28px (h-7 w-7) — established pattern from existing `Button variant="ghost" className="h-7 w-7 p-0"` usage; no change this phase

## Color

All values are from `src/globals.css` CSS custom properties (both `:root` and `.dark` are identical, so the palette is dark-only by design).

| Role | CSS Variable | HSL Value | Usage |
|------|-------------|-----------|-------|
| Dominant (60%) | `--background` | hsl(160 38% 16%) | Board surface — the table felt |
| Secondary (30%) | `--card` | hsl(160 38% 10%) | Header band (opponents + controls); `bg-card` removed from personal spread band (POLISH-03) |
| Zone fill | `--secondary` | hsl(160 30% 22%) | Pile zone and spread zone backgrounds (`bg-secondary`) |
| Accent (10%) | `--primary` | hsl(38 92% 50%) | Reserved for: active drop target borders (`border-primary`), selection rings (`ring-primary/30`), selection count badge background, dashed drop strip highlight (ZONE-01) |
| Muted text | `--muted-foreground` | hsl(220 9% 46%) | Pile labels, zone labels, drag-origin placeholder borders |
| Destructive | `--destructive` | hsl(0 84% 60%) | Destructive actions only (none in this phase) |
| Border default | `--border` | hsl(160 20% 20%) | All non-active zone borders |

Accent (`--primary`, amber) is reserved for **exactly these elements**:
1. Drop target zone border when a dragged card is over the zone (`isOver === true`)
2. Card selection ring (`ring-primary/30`)
3. Selection count badge background
4. Slim drop-target strip border (ZONE-01, `border-primary border-dashed`)

## Interaction Contracts

### POLISH-01 — Remove empty zone body text

- **Before:** `SpreadZone` renders `<span className="text-xs text-muted-foreground">{pile.name}</span>` inside the zone container when `isEmpty` (SpreadZone.tsx line 176).
- **After:** Remove the `{isEmpty ? (<span>…</span>) : …}` branch. Empty zone shows only its dashed border; no interior text.
- State: only affects visual empty state. No interaction change.

### POLISH-02 — Pile controls header row

- **Before:** `PileZone` renders label above the pile card, then a separate `<div className="flex gap-1 mt-1">` for controls below the card.
- **After:** Single `flex justify-between items-center` header row containing:
  - Left: `<span className="text-xs text-muted-foreground">{pile.name}</span>` — hidden at narrow widths via `hidden sm:inline`
  - Right: `<div className="flex gap-1">` with Eye/Shuffle/SelectAll buttons
- The outer wrapper changes from `flex flex-col items-center gap-1` to `flex flex-col gap-1` (remove `items-center` so header row spans full pile width, not centered to card).
- Buttons remain `Button variant="ghost" className="h-7 w-7 p-0"` — no style change.
- At mobile widths (`< sm` breakpoint), the label is hidden; only the 3 icon buttons appear.

### POLISH-03 — Personal spread band

- **Before:** `<div className="flex-shrink-0 bg-card px-4 py-2">` wraps the personal SpreadZone in BoardView.tsx lines 90–103.
- **After:** Remove `bg-card`. Reduce vertical padding from `py-2` (8px) to `py-1` (4px). Result: `<div className="flex-shrink-0 px-4 py-1">`.
- The spread band now has `bg-background` inherited from the board body — it visually merges into the table surface.
- No change to SpreadZone component internals.

### POLISH-04 — Compact zone heights

- Card zone height tokens: mobile `h-[64px]` (was `h-[79px]`), desktop `sm:h-[112px]` → `sm:h-[90px]`.
- Applies to the `className` on the droppable `div` inside `PileZone.tsx` (line 54) and `SpreadZone.tsx` (line 169).
- `GridZone.tsx` must be audited for matching height changes to stay proportional (executor must check its height declarations).
- Inter-zone gap in the center flex row (`gap-4` in BoardView.tsx line 74) is unchanged — only zone heights contract.

### ZONE-01 — Empty personal spread collapse/reveal

- **Collapsed state (default, empty zone):** The spread zone wrapper renders with `height: 0` and `overflow: hidden`. The droppable node remains mounted in the DOM (dnd-kit requires the node for collision detection).
- **Revealed state (drag hovering over zone area):** When `isOver === true` on the spread's `useDroppable`, the wrapper expands to the slim strip dimensions: `h-[40px] sm:h-[56px]`.
- **Visual:** Slim strip uses `border border-dashed border-primary rounded-lg` — same visual language as empty pile drop targets when `isOver`.
- **Implementation note:** Zero-height collapse may cause dnd-kit proximity detection to fail. The planner must verify collision detection against a `height: 0` node and propose a fallback (e.g., `h-px opacity-0` with a larger invisible hit area) if needed.
- **Not triggered by:** drag start (CONTEXT.md D-05/D-06 — the strip only appears on `isOver`, not during a drag in general).
- **When zone has cards:** Normal `SpreadZone` renders at full `h-[64px] sm:h-[90px]` — collapse logic applies only when `isEmpty === true`.
