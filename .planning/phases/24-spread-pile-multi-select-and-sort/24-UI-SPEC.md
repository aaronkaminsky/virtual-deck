## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, badge offset |
| sm | 8px | Cell gap, button gap |
| md | 16px | Grid outer padding, zone label gap |
| lg | 24px | Section padding |
| xl | 32px | Layout gaps |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing |

Exceptions:
- Card dimensions are fixed at 42×59px (mobile) / 63×88px (desktop) — set by CardFace/CardBack; not on the 8-point scale, do not change.
- Count badge: positioned `absolute -bottom-2 -right-2` (8px offset from card corner) — inherited from PileZone pattern.
- Grid cell height: 79px mobile / 112px desktop — matches card height plus 1px border (existing zone sizing).
- Touch targets (zone controls): minimum 28×28px (`h-7 w-7`) — inherited from existing Button ghost pattern.

## Component Inventory

New component: `GridZone`

Replaces `<SpreadZone>` for the communal pile (`id='play'`) only. Personal spread zones continue to use `SpreadZone` unchanged.

### GridZone structure

```
<div> (zone wrapper — flex-col gap-1)
  <div> (header row — flex items-center)
    <span text-xs text-muted-foreground> (zone label: "Play Area")
    <span text-xs bg-primary text-primary-foreground rounded-full px-1.5> (selection count pill — only when multi-select active, out of scope this phase)
  </div>
  <div data-testid="grid-zone-play"> (CSS grid container)
    [2×N GridCell components]
  </div>
  <div> (controls row — flex gap-1)
    <Button variant="ghost" h-7 w-7 p-0> (face toggle — Eye/EyeOff icon)
  </div>
</div>
```

### GridCell structure

Each cell is an independent `useDroppable` target:

```
<div ref={setNodeRef}> (cell container — relative, fixed dimensions)
  {occupied and not dragging source}:
    <DraggableCard or CardFace/CardBack> (top card rendered here)
    <Badge absolute -bottom-2 -right-2> (×N count — only when stack size > 1)
  {dragging source card from this cell}:
    <div absolute inset-0 rounded-md border-2 border-dashed border-muted-foreground> (drag placeholder)
  {empty}:
    (nothing — empty cell shows bg-secondary with dashed border)
</div>
```

### Grid layout

| Breakpoint | Columns | Rows | Cell width | Cell height |
|------------|---------|------|------------|-------------|
| Mobile (<640px / below `sm`) | 4 | 2 | 56px (w-14) | 79px (h-[79px]) |
| Desktop (sm+) | 7 | 2 | 80px (w-20) | 112px (h-[112px]) |

Implementation: `grid grid-cols-4 sm:grid-cols-7 gap-1` with fixed-dimension cells. No `overflow-x-auto` — grid is fixed-width, always two rows.

Source: D-01 (CONTEXT.md).

### Count badge

- Component: `<Badge>` from `@/components/ui/badge` (shadcn — same component used in PileZone)
- Position: `absolute -bottom-2 -right-2`
- Content: `×{count}` (e.g. `×3`) when stack contains more than 1 card
- Hidden when cell has exactly 1 card
- Source: D-10 (CONTEXT.md); pattern from PileZone.tsx line 80

### Drag source placeholder

When the card being dragged originated from a grid cell, that cell shows:
- `<div className="absolute inset-0 rounded-md border-2 border-dashed border-muted-foreground" />`
- Same pattern used in SpreadZone.tsx (draggingCardId) and PileZone.tsx (isDraggingTopCard)

### Cell drop highlight

When a drag is active and the pointer is over a cell (`isOver === true` from `useDroppable`):
- Cell border: `border-primary` (replaces `border-border`)
- No background change — border-only highlight, consistent with existing SpreadZone `isOver` pattern

## Interaction Contract

### Drag into grid from outside (hand or pile)

1. Player begins drag from hand or pile.
2. As pointer enters grid, the cell under pointer becomes the active drop target (`isOver: true`) — its border switches to `border-primary`.
3. Only one cell is highlighted at any time (D-03).
4. On drop: card is added to that cell's position (row, col). If cell is occupied, card stacks on top (D-08). No rejection dialog.
5. Server receives `MOVE_CARD` or `PLAY_CARD_SET` action; server assigns `gridPositions[cardId] = { row, col }`.

### Drag within grid (cell to cell)

1. Player drags top card from a source cell (D-11 — top card only).
2. Source cell shows dashed placeholder while drag is in flight.
3. Hovered target cell highlights with `border-primary`.
4. On drop: `MOVE_GRID_CARD { cardId, pileId, toRow, toCol }` action dispatched.
5. Server moves `gridPositions[cardId]` to new coordinates and broadcasts `STATE_UPDATE`.

### Drag out of grid (to hand or pile)

Uses existing drag infrastructure — card leaves grid, server clears `gridPositions[cardId]` on move.

### Multi-card selection within grid

Out of scope (D-12). Select-All button is not present on `GridZone` in this phase.

### Face toggle

Retained from SpreadZone: Eye/EyeOff ghost button, 28×28px, dispatches `SET_PILE_FACE`.
