## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding |
| sm | 8px | Compact element spacing |
| md | 16px | Default element spacing |
| lg | 24px | Section padding |
| xl | 32px | Layout gaps |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing |

Exceptions:
- Faint strip resting height: `h-4` (16px) — ¼ of full zone height `h-[64px]`
- Full spread zone resting height: `h-[64px]` desktop / `h-[88px]` sm
- Spread zone isOver height: `h-[40px]` desktop / `h-[56px]` sm
- OpponentHand: no size expansion at drag-start or drag-over (D-02, D-04 from CONTEXT.md)

## Color

| Role | CSS Token | Computed Value | Usage |
|------|-----------|----------------|-------|
| Dominant (60%) | `--background` | `hsl(160 38% 16%)` | Board surface, dark felt |

- At drag-start size expansion on OpponentHand (removes `min-h-[44px] min-w-[80px]` — D-02)

## Component Interaction Contract

### CTRL-06 — OpponentHand drop-target border

**File:** `src/components/OpponentHand.tsx` lines 31–38

| State | className |
|-------|-----------|
| Idle (no drag) | `border-2 border-transparent` |
| Drag active, not hovering | `border-2 border-transparent` (same as idle — no dashed state) |
| `isOver = true` | `border-2 border-primary` |

Implementation: replace the three-branch ternary with a two-branch ternary:
`isOver ? 'border-2 border-primary' : 'border-2 border-transparent'`

Remove: `dragIsActive && 'min-h-[44px] min-w-[80px]'` class line entirely.

Retain: `dragIsActive && cardCount === 0` "Drop to pass" text hint (line 65) — text-only drag-start feedback when zone is empty.

`useDndContext` import: keep (still needed for `dragIsActive` on line 65 text hint).

### LAYOUT-06 — SpreadZone empty resting state

**File:** `src/components/SpreadZone.tsx` lines 166–174

| State | className |
|-------|-----------|
| `isEmpty && interactive !== false` (resting) | `h-4 border border-dashed border-muted-foreground/30 rounded-md` |
| `isEmpty && interactive !== false && isOver` | `min-w-[56px] sm:min-w-[80px] h-[40px] sm:h-[56px] border border-dashed border-primary rounded-lg flex items-center px-2` |
| Not empty | `min-w-[56px] h-[64px] sm:min-w-[80px] sm:h-[88px] rounded-lg border flex items-center px-2 overflow-x-auto bg-secondary` (unchanged) |

Replace `'h-px opacity-0'` with `'h-4 border border-dashed border-muted-foreground/30 rounded-md'`.

Controls guard (`interactive !== false && !isEmpty`) is unchanged — controls remain hidden when empty. No new controls introduced.
