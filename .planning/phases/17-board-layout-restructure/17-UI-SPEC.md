## Spacing Scale

All values are multiples of 4. Derived from existing Tailwind usage in BoardView, HandZone, and SpreadZone.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, zone label to zone edge (`gap-1`) |
| sm | 8px | Compact element spacing within rows (`gap-2`) |
| md | 16px | Default horizontal padding for all zone rows (`px-4`) |
| lg | 24px | Inter-zone horizontal gaps in opponent row and center row (`gap-6`) |
| xl | 32px | — |
| 2xl | 48px | — |
| 3xl | 64px | — |

Exceptions:
- Card overlap offset: `-20px` (`-ml-5`) — intentional negative margin for fan layout; not a spacing token
- Hand zone height: 128px (fixed, preserves full card + selection lift clearance)
- Spread zone height: 112px (fixed, preserves card height 88px + padding)
- Communal zone: `flex-1` width within center row — grows to fill remaining space after pile zones

## Layout Contract

This is the primary deliverable of Phase 17. The contract below defines the five horizontal bands in `BoardView.tsx`.

### Five-Band Vertical Structure

```
┌─────────────────────────────────────────────────────┐
│  BAND 1: Header                                     │
│  bg-card | px-4 py-2 | flex row, space-between      │
│  Left: opponent columns (hand + spread, scrollable) │
│  Right: Copy Link button + ControlsBar              │
├─────────────────────────────────────────────────────┤
│  BAND 2: Center Row                                 │
│  flex-1 | flex row | items-center | px-4 | gap-4    │
│  Left: pile zone(s) — fixed width, flex-shrink-0    │
│  Center: communal spread zone — flex-1 min-w-0      │
├─────────────────────────────────────────────────────┤
│  BAND 3: Player Spread Zone                         │
│  bg-card | px-4 py-2 | full-width SpreadZone        │
├─────────────────────────────────────────────────────┤
│  BAND 4: Player Hand Zone                           │
│  bg-card | h-[128px] | HandZone (unchanged)         │
└─────────────────────────────────────────────────────┘
```

Note: The ConnectionBanner sits above Band 1 as a non-band overlay strip when active.

### Band 1: Header (Opponents + Controls)

- `flex items-center justify-between px-4 py-2 gap-4 bg-card`
- Left side: `flex items-start gap-4 flex-1 overflow-x-auto` — opponent columns scroll horizontally if > 4 players
- Each opponent column: `flex flex-col gap-1` — OpponentHand (label + card count) stacked above their SpreadZone
- Right side: Copy Link button + ControlsBar — unchanged from current implementation (Phase 18 handles collapse)

### Band 2: Center Row (Piles + Communal Zone)

- `flex-1 flex items-center px-4 gap-4` — takes all remaining vertical space between header and player bands
- Pile zone(s): `flex-shrink-0` — natural card-stack width, do not grow
- Communal zone: `flex-1 min-w-0` wrapper around `SpreadZone` — zone label + spread container fill center
- Communal zone spread container: `w-full h-[112px]` — inherits full flex-1 width

Communal zone is visually dominant. It is the centerpiece of the board. Pile zones are subordinate flanks.

### Band 3: Player Spread Zone

- `bg-card px-4 py-2` — separate horizontal band, full board width
- Single `SpreadZone` for `mySpreadZone`
- Zone label (`text-xs text-muted-foreground`) above container, Face up/down toggle below

### Band 4: Player Hand Zone

- Unchanged: `HandZone` component with `bg-card h-[128px]`
- Player label row above, cards row, selection badge in label row when ≥2 selected

### Vertical Proportion Target (1080p)

At 1080p (typical board height ~960px after browser chrome):

| Band | Approximate Height | Sizing |
|------|--------------------|--------|
| Header | ~72px | auto (content-driven) |
| Center Row | ~720px | flex-1 |
| Player Spread Zone | ~152px | auto (card height 112px + label + button) |
| Player Hand Zone | ~128px | fixed |

All zones visible simultaneously. No scrolling required. Total ~1072px — fits 1080p with browser chrome.

## Interaction Contract

### Drag Behavior (SPREAD-04 fix)

- After removing `DraggableCard` from `SortableSpreadCard`, the single dnd-kit ID per card is the `useSortable` ID (`card.id`)
- `useSortable` data shape is unchanged: `{ card, fromZone: 'pile', fromId: pileId, toZone: 'pile', toId: pileId }`
- `BoardDragLayer.tsx` reads `active.data.current` — verify it handles `fromZone: 'pile'` from sortable data (not only from `useDraggable` data)
- `PileZone.tsx` drop target reads `fromZone` from incoming drag data — verify compatibility
- No behavior change for the end user: drag-out and sort-within still work identically

### Selection State (preserved, not changed)

- `SortableSpreadCard` does not currently support selection (Phase 20 work)
- Phase 17 must not accidentally wire `onToggleSelect` into spread cards
- The `selectedIds`/`onToggleSelect` props stay scoped to `HandZone` only

### Click vs. Drag Disambiguation

`SortableSpreadCard` must adopt the `didDragRef` pattern from `SortableHandCard`:
- `onPointerDown`: set `didDragRef.current = false`
- `onDragStart` / movement detection: set `didDragRef.current = true`
- `onClick`: only fire if `!didDragRef.current`
- Currently SpreadZone has no click handler on cards — this is pre-work so Phase 20 can safely add `onToggleSelect` without click-drag conflicts
