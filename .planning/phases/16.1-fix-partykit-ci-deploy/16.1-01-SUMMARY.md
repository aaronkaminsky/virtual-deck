## User Setup Required

**PARTYKIT_TOKEN secret must be created manually (D-04, one-time setup).**

Steps:

1. Mint a token locally:
   ```
   npx partykit login        # if not already logged in
   cat ~/.partykit/config.json   # copy the access_token field
   ```
   (Or try `npx partykit token` if that command is available on your version.)

2. Open GitHub: `https://github.com/<your-account>/virtual-deck/settings/secrets/actions`

3. Click **New repository secret**

4. Name: `PARTYKIT_TOKEN` (exact, case-sensitive)

5. Value: paste the token

6. Click **Add secret**

7. Confirm it appears under **Repository secrets** (NOT environment secrets) — D-04 requires repo-level scope.

After creating the secret, push `deploy.yml` to main and verify the CI run (Task 2 — see checkpoint below).
