## Color

Inherited from Phase 3 — all CSS variables unchanged:

| Role | CSS Variable | Resolved HSL | Usage |
|------|-------------|--------------|-------|
| Dominant (60%) | `--background` | hsl(160 38% 16%) | Board felt surface, full-viewport background |
| Secondary (30%) | `--card` / `--secondary` | hsl(160 38% 10%) / hsl(160 30% 22%) | Pile zones, top strip background, popover background |
| Accent (10%) | `--primary` | hsl(38 92% 50%) — amber/gold | Active game phase indicator, Deal button (setup phase only), focused number input ring |
| Destructive | `--destructive` | hsl(0 84% 60%) | Reset button, Reset confirmation dialog destructive action button |

Accent reserved for:
1. Deal button fill (setup phase) — the single primary CTA during setup
2. Active game phase badge border or background
3. Focus ring on Deal popover number input (`--ring` is already amber)
4. Drag-over drop highlight borders (carried forward from Phase 3)

Do NOT use accent on: Undo button, Shuffle pile button, per-pile face-toggle button, or any play-phase controls. Play-phase controls use secondary/muted styling.

Destructive reserved for:
1. Reset table button — `variant="destructive"` shadcn Button
2. "Reset table" confirmation dialog confirm button — `variant="destructive"`

Disabled state:
- Undo button when no undo available: shadcn Button `disabled` prop — inherits `opacity-50 cursor-not-allowed`. No custom styling.

## Layout Contract

### Top Strip (modified from Phase 3)

The existing opponent strip is extended to become a full header strip containing opponent hands on the left and controls on the right:

```
┌──────────────────────────────────────────────────────────────┐
│  [Player] ♠♠♠♠  [Player] ♠♠♠♠       [Setup: Deal ▼]        │
│  opponent hands (flex-1, overflow-x-auto)   controls (flex-0) │
└──────────────────────────────────────────────────────────────┘
```

- Top strip total height: `h-[88px]` — unchanged from Phase 3
- Layout: `flex items-center justify-between px-4 gap-4 bg-card`
- Left region: opponent hands — `flex items-center gap-2 flex-1 overflow-x-auto`
- Right region: ControlsBar — `flex items-center gap-2 flex-shrink-0`

Source: CONTEXT.md D-13, D-14.

### ControlsBar Layout

Setup phase (Deal visible, Undo + Reset hidden):
```
[Deal ▼]
```

Play phase (Deal hidden, Undo + Reset visible):
```
[↩ Undo]  [Reset]
```

- Buttons use shadcn `size="sm"` (height: 36px)
- Gap between buttons: 8px (`gap-2`)
- Deal button: `variant="default"` (filled amber — primary accent) with ChevronDown icon from lucide-react
- Undo button: `variant="outline"` with Undo2 icon from lucide-react; disabled when no prior move
- Reset button: `variant="destructive"` with RotateCcw icon from lucide-react

Source: CONTEXT.md D-07, D-08, D-11.

### Deal Popover

Triggered by Deal button. Uses shadcn Popover:

```
┌─────────────────────┐
│  Cards per player   │
│  [  3  ] [−] [+]   │
│        [Deal]       │
└─────────────────────┘
```

- Popover width: 200px fixed (`w-[200px]`)
- Padding: 16px (`p-4`)
- Number input: shadcn Input, `type="number"`, `min="1"`, `max` = floor(draw_pile_count / player_count)
- Confirm button: shadcn Button `variant="default"`, full width, label "Deal"
- Popover opens below the Deal button: `side="bottom" align="end"`

Source: CONTEXT.md D-07, D-09.

### Reset Confirmation Dialog

Uses shadcn AlertDialog:

```
┌──────────────────────────────────┐
│  Reset table?                    │
│                                  │
│  All cards return to the draw    │
│  pile and will be reshuffled.    │
│  This can't be undone.           │
│                                  │
│       [Cancel]  [Reset table]    │
└──────────────────────────────────┘
```

- Dialog title: "Reset table?" — 16px, weight 600
- Dialog body: 14px, weight 400, `text-muted-foreground`
- Cancel button: `variant="outline"`
- Confirm button: `variant="destructive"`, label "Reset table"

Source: CONTEXT.md D-15.

### Per-Pile Shuffle Button (PileZone modification)

The existing PileZone has a face-toggle button. Shuffle is added alongside it:

```
┌──────────────────┐
│                  │
│   [card face]    │
│                  │
│ [↕ Face] [⇄ Shuf]│
└──────────────────┘
```

- Both buttons: shadcn Button `variant="ghost"`, `size="xs"` / explicit `h-7 px-2 text-xs`
- Icon: `Shuffle` from lucide-react (for shuffle), existing face icon carried forward
- Buttons sit in a `flex gap-1` row at the bottom of the PileZone container
- Available in both setup and playing phases (CONTEXT.md D-12)

## Interaction Contract

### Card Flip (CARD-03)

- Target: cards in pile zones only. Hand cards are not clickable.
- Trigger: single click (`onClick`) on `DraggableCard` when rendered inside a `PileZone` context.
- Click guard: if `isDragging` is true at the time of the click event, suppress the flip action. Use a ref that is set to true on `onDragStart` and false after `onDragEnd` settles (same ref pattern as socket buffer in Phase 3).
- Visual: no hover overlay, no indicator. The card itself is the click target. After flip, card re-renders with new face/back state from server `STATE_UPDATE`.
- No local optimistic update — wait for server broadcast before re-render.

Source: CONTEXT.md D-01, D-02, D-03.

### Pass Card (CARD-04)

- Drag source: `DraggableCard` in `HandZone` — already draggable via dnd-kit.
- Drop target: `OpponentHand` zones — add `useDroppable` with `id: "opponent-hand-{playerId}"`.
- Drop highlight: `OpponentHand` shows `--primary` (amber) border when `isOver === true` — same pattern as PileZone drop highlight.
- Opponent hand zone size: maintain existing `h-[88px]` strip — no size change needed to accept drops.
- Server validates sender; if invalid, card snaps back (same silent snap-back pattern as Phase 3).

Source: CONTEXT.md D-04, D-05, D-06.

### Undo (CTRL-04)

- Button state: disabled (`disabled` prop) until player performs a card action this session. After one undo, returns to disabled until next action.
- No confirmation required — undo is reversible by re-doing the move manually.
- No visual countdown or timer — single-step only.
- After undo fires, server broadcasts `STATE_UPDATE`. The Undo button immediately disables (optimistic — do not wait for broadcast).

Source: CONTEXT.md D-17, D-18, D-19.

### Phase Transition (Setup → Playing)

- Deal action fires → server broadcasts updated state with `phase: "playing"`.
- ControlsBar re-renders: Deal button unmounts, Undo + Reset mount.
- Transition is immediate on state update — no animation needed.
- No explicit "you are now playing" toast or notification.

Source: CONTEXT.md D-10, D-11.
