## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, card cascade inner spacing |
| sm | 8px | Zone label margin, button padding |
| md | 16px | Section horizontal padding (px-4) |
| lg | 24px | Spread row gap between zones (gap-6) |
| xl | 32px | Middle piles row gap (gap-6 = 24px, gap-8 = 32px — use gap-6) |
| 2xl | 48px | Not used in this phase |
| 3xl | 64px | Not used in this phase |

Exceptions:
- Card cascade overlap: `-20px` left margin per card after the first (`-ml-5`). This is a visual layout value derived from card width (63px), not a spacing token.
- SpreadZone container height: `112px` — matches PileZone height exactly. Source: PileZone.tsx `h-[112px]`.
- SpreadZone container min-width: `80px` — matches PileZone width. Expands with card content.
- HandZone height: `128px` — unchanged, existing constraint.
- Touch target for toggle button: `h-7` (28px) — consistent with PileZone existing button height.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `hsl(160 38% 16%)` — `--background` | Board surface; SpreadZone sits on this surface |
| Secondary (30%) | `hsl(160 30% 22%)` — `--secondary` | SpreadZone container background (`bg-secondary`), card zone bg |

Accent reserved for: drop target active border on SpreadZone (`isOver ? 'border-primary' : 'border-border'`). This is the same reservation used in PileZone. Accent/primary is NOT used for zone labels, empty state text, or toggle buttons.

Additional semantic colors used:
- `--border` (`hsl(160 20% 20%)`): resting SpreadZone border; dashed variant for empty state
- `--muted-foreground` (`hsl(220 9% 46%)`): zone labels, empty state text, toggle button text
- `--card` (`hsl(160 38% 10%)`): HandZone strip background (unchanged)

## Component Inventory

New components and modifications this phase introduces:

### New: `SpreadZone` (`src/components/SpreadZone.tsx`)

**Visual anatomy:**

```
[zone label text — displayName or "Communal"]        ← text-xs text-muted-foreground, above container
┌─────────────────────────────────────────────────┐  ← border-dashed (empty) / border-border (cards)
│  [CardBack or DraggableCard] [-ml-5 overlap...] │  ← bg-secondary, h-[112px], min-w-[80px]
└─────────────────────────────────────────────────┘  ← border-primary when isOver (drop target active)
[Face up / Face down]                                ← Button variant="ghost" h-7 px-2 text-xs
```

**States:**

| State | Visual |
|-------|--------|
| Empty (default) | `border-dashed border-border bg-secondary`, label text centered inside container |
| Has cards | Solid `border-border bg-secondary`, cascading DraggableCard row with `-ml-5` overlap after first card |
| Drop target active (isOver) | `border-primary` border (2px solid amber/gold) |
| Face-down zone | Cards arrive as `MaskedCard` (no id field); render `CardBack` component; only top card is draggable |
| Face-up zone (default) | All cards render as `DraggableCard` with full rank/suit; all cards individually draggable |

**Card cascade layout:**
- Container: `flex items-center overflow-x-auto px-2`
- Cards: `flex items-center` row; each card after the first: `-ml-5` (−20px left margin)
- Card size: 63px wide × 88px tall — unchanged from all other zones
- Overflow: `overflow-x-auto` on the inner card row; zone container height is fixed at 112px

**Drop target registration:**
- `useDroppable` id: `` `pile-${pile.id}` `` (e.g., `pile-spread-communal`, `pile-spread-{playerToken}`)
- data: `{ toZone: 'pile' as const, toId: pile.id }`
- Reason: Must start with `pile-` for BoardDragLayer.tsx customCollision to include it in pointerWithin detection

**data-testid convention:**
- `data-testid={`spread-zone-${pile.id}`}` on the drop container div
- Examples: `spread-zone-spread-communal`, `spread-zone-spread-abc123`

**Toggle button:**
- Label: "Face up" when `pile.faceUp !== false`; "Face down" when `pile.faceUp === false`
- Title attribute: "Cards land face-up (click to flip)" / "Cards land face-down (click to flip)"
- Dispatches: `SET_PILE_FACE` action — same as PileZone
- No Shuffle button on SpreadZone (shuffle is only for pile-region zones)

### Modified: `BoardView` (`src/components/BoardView.tsx`)

**Layout change: 3 sections → 4 sections**

```
h-screen w-screen overflow-hidden flex flex-col bg-background
  ├── [ConnectionBanner]
  ├── Header (auto-height, was h-[104px]) — remove fixed height, add py-2
  │    ├── flex items-start gap-4 flex-1 overflow-x-auto  ← opponents
  │    │    └── [per opponent] flex flex-col gap-1
  │    │         ├── OpponentHand (unchanged)
  │    │         └── SpreadZone (their personal zone, always visible if connected)
  │    └── Copy link + ControlsBar (unchanged, right side)
  ├── Middle (flex-1) — PileZone × 3 (draw, discard, play ONLY)
  ├── Spread Row (auto-height, py-2 px-4) — new section
  │    ├── SpreadZone spread-communal
  │    └── SpreadZone spread-{myPlayerId}
  └── HandZone (unchanged, h-[128px])
```

**Pile filtering in BoardView:**

```typescript
const pilePiles = gameState.piles.filter(p => (p.region ?? 'pile') === 'pile');
const spreadPiles = gameState.piles.filter(p => p.region === 'spread');
const mySpreadZone = spreadPiles.find(p => p.id === gameState.myPlayZoneId);
const communalZone = spreadPiles.find(p => p.id === 'spread-communal');
```

**Spread row layout class:** `flex items-start gap-4 px-4 py-2`

**Opponent zone visibility rule:** Opponent SpreadZone renders when `opponentSpread` is found in `spreadPiles`. A connected player's zone exists immediately after their first connect; guard with `{opponentSpread && <SpreadZone ... />}`.

**Header height:** Remove `h-[104px]`. Use `py-2` padding. Header grows to accommodate opponent hand column (OpponentHand ~59px card height + labels + SpreadZone 112px + labels = ~190px in 2-player game). This is expected and correct.

## Interaction Contracts

### Drag and Drop

| Interaction | Behavior |
|-------------|----------|
| Drag card from hand to SpreadZone | MOVE_CARD dispatched; if zone non-empty, existing Top/Bottom/Random dialog appears from BoardDragLayer |
| Drag card from PileZone to SpreadZone | Same as above |
| Drag card from SpreadZone to anywhere | DraggableCard with `fromZone: "pile" fromId: pile.id`; existing MOVE_CARD routing handles it unchanged |
| Drag card from SpreadZone to SpreadZone | Works via existing MOVE_CARD; inserts at chosen position |
| Drop target highlight | `border-primary` (amber) when pointer is inside zone rect during drag |
| No hover state without active drag | Zone always shows `border-border` (resting state) when no drag active |

Note on insert position dialog: When a card is dropped on a non-empty SpreadZone, `BoardDragLayer.handleDragEnd` shows the Top/Bottom/Random dialog (same as PileZone). This is intentional for v1.4. The RESEARCH.md open question about bypassing the dialog for spread zones is deferred — accept dialog behavior for this phase.

### Face Toggle

| Trigger | Behavior |
|---------|----------|
| Click "Face up" button | Dispatches `SET_PILE_FACE { pileId, faceUp: false }` |
| Click "Face down" button | Dispatches `SET_PILE_FACE { pileId, faceUp: true }` |
| Any player can toggle any zone | No ownership restriction (D-16) |

### Empty State

SpreadZone with zero cards:
- Container: `border-dashed border-border bg-secondary h-[112px] min-w-[80px]`
- Interior: Zone label text centered — same text as the external label above the container
- No icon, no call-to-action — purely a passive drop target
