<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| server STATE_UPDATE → client | ClientGameState payload crosses here. Client must tolerate pre-migration payloads where `pile.region` / `pile.ownerId` / `gameState.myPlayZoneId` may be undefined or empty (STATE_UPDATE arrives AFTER first onConnect; there is a brief gap). |
| user drag → client → server MOVE_CARD | Drag gesture produces client-issued MOVE_CARD. Server handles validation; client passes fromZone="pile" fromId=pile.id unchanged. |
| user click → SET_PILE_FACE | Any user can toggle any zone per D-16. No client-side gating. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-14-06 | Information Disclosure | SpreadZone rendering masked (face-down non-top) cards | mitigate | MUST use `'id' in card` guard per card; masked cards render as `<CardBack />` and are NOT wrapped in `<DraggableCard>` (which would require a card id that does not exist on MaskedCard). Prevents runtime errors and accidental reveal of id/suit/rank props. |
| T-14-07 | Tampering | Client filters piles by region for layout | accept | Client-side region filter (`p.region ?? 'pile'`) is for UX layout only — it does not grant authority. Server remains source of truth; any pile id is a valid MOVE_CARD target. Defense in depth: server validates pile exists. |
| T-14-08 | Denial of Service | BoardView renders before first STATE_UPDATE populates myPlayZoneId | mitigate | Guard with `{mySpreadZone && <SpreadZone ... />}` — identical pattern to existing `player?.displayName ?? ''` guards. Prevents undefined-pile runtime errors during the connect → first-state-update window. |
| T-14-09 | Spoofing | Playwright e2e test relies on data-testid values that could collide with user-chosen names | mitigate | SpreadZone testid is `` `spread-zone-${pile.id}` `` where pile.id is server-computed (`spread-${playerToken}`, playerToken bounded 64 chars). Communal zone id is the literal string `spread-communal`. No player-supplied strings enter the testid. |
</threat_model>

    ```typescript
    import { useDroppable } from '@dnd-kit/core';
    import type { Card, ClientPile, ClientAction } from '@/shared/types';
    import { Button } from '@/components/ui/button';
    import { DraggableCard } from './DraggableCard';
    import { CardBack } from './CardBack';
    import { cn } from '@/lib/utils';

    interface SpreadZoneProps {
      pile: ClientPile;
      sendAction: (action: ClientAction) => void;
      draggingCardId: string | null;
    }

    export function SpreadZone({ pile, sendAction, draggingCardId: _draggingCardId }: SpreadZoneProps) {
      const { setNodeRef, isOver } = useDroppable({
        id: `pile-${pile.id}`,
        data: { toZone: 'pile' as const, toId: pile.id },
      });

      function handleToggleFace() {
        sendAction({ type: 'SET_PILE_FACE', pileId: pile.id, faceUp: !pile.faceUp });
      }

      const isEmpty = pile.cards.length === 0;

      return (
        <div className="flex flex-col gap-1">
          <span className="text-xs text-muted-foreground">{pile.name}</span>
          <div
            ref={setNodeRef}
            data-testid={`spread-zone-${pile.id}`}
            className={cn(
              'min-w-[80px] h-[112px] rounded-lg border flex items-center px-2 overflow-x-auto bg-secondary',
              isEmpty ? 'border-dashed' : '',
              isOver ? 'border-primary' : 'border-border'
            )}
          >
            {isEmpty ? (
              <span className="text-xs text-muted-foreground">{pile.name}</span>
            ) : (
              <div className="flex items-center">
                {pile.cards.map((card, i) => (
                  <div
                    key={'id' in card ? (card as Card).id : `masked-${i}`}
                    className={cn('flex-shrink-0', i > 0 ? '-ml-5' : '')}
                  >
                    {'id' in card ? (
                      <DraggableCard card={card as Card} fromZone="pile" fromId={pile.id} />
                    ) : (
                      <CardBack />
                    )}
                  </div>
                ))}
              </div>
            )}
          </div>
          <Button
            variant="ghost"
            className="h-7 px-2 text-xs"
            onClick={handleToggleFace}
            title={pile.faceUp !== false ? 'Cards land face-up (click to flip)' : 'Cards land face-down (click to flip)'}
          >
            {pile.faceUp !== false ? 'Face up' : 'Face down'}
          </Button>
        </div>
      );
    }
    ```
