## Spacing Scale

Declared values (multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, rank/suit inline padding within card face |
| sm | 8px | Gap between cards in hand row, pile zone internal padding |
| md | 16px | Board section padding, hand strip horizontal padding |
| lg | 24px | Gap between pile zones, opponent strip vertical padding |
| xl | 32px | Gap between board regions (opponent strip, pile row, hand strip) |
| 2xl | 48px | Not used in Phase 3 |
| 3xl | 64px | Not used in Phase 3 |

Exceptions:
- Card face dimensions: 63px wide × 88px tall (standard poker card ratio — not a spacing token, a fixed dimension). Source: RESEARCH.md Pattern 4.
- Drag ghost scale: `transform: scale(1.07)` — not spacing, a visual lift indicator. Source: CONTEXT.md D-16.
- Hand strip height: 120px–140px fixed. Opponent strip height: 80px–100px fixed. Source: CONTEXT.md specifics section.

|------|-------------|--------------|-------|
| Dominant (60%) | `--background` | hsl(160 38% 16%) | Board felt surface, full-viewport background |
| Secondary (30%) | `--card` / `--secondary` | hsl(160 38% 10%) / hsl(160 30% 22%) | Pile zone areas, opponent strip background, hand strip background |
| Accent (10%) | `--primary` | hsl(38 92% 50%) — amber/gold | Drag-over highlight on valid drop targets, pile zone active border, pile card count badge |
| Destructive | `--destructive` | hsl(0 84% 60%) | Not used in Phase 3 (no destructive actions) |
| Card face background | #ffffff (hardcoded) | — | Card face only — intentional contrast against dark felt |
| Red suit color | #dc2626 (Tailwind red-600) | — | Hearts and diamonds rank/suit text |
| Black suit color | #111827 (near-black) | — | Spades and clubs rank/suit text |
| Card back base | #1a3050 (hardcoded) | — | CSS crosshatch card back base color |
| Card back lines | #1e3a5f (hardcoded) | — | Crosshatch gradient line color |

Accent reserved for:
1. Pile zone drop-over highlight border (when card is dragged over a valid pile zone)
2. Hand strip drop-over highlight border (when card is dragged over the player's own hand)
3. Pile card count Badge component background
4. `--ring` focus ring (already set to amber in globals.css — inherited behavior)

Do NOT use accent on: card faces, card backs, opponent strips, pile labels, or general UI chrome.

Source: `src/globals.css` for all HSL values. CONTEXT.md D-02 for card face colors. RESEARCH.md Pattern 4 for card back colors.

## Interaction Contract

### Drag-and-Drop States

Each draggable card must have exactly 3 visual states:

| State | Visual Treatment |
|-------|-----------------|
| Default (not dragging) | Full opacity, no shadow |
| Active drag source (ghost visible) | opacity: 0 on original card element; `DragOverlay` shows full clone at scale(1.07) |
| Drag-over valid drop target | Target zone shows `--primary` (amber) border highlight, no other change |

Drop target zones:
- PileZone: `useDroppable` — highlights on `isOver`
- HandZone: `useDroppable` — highlights on `isOver`
- Opponent hand: NOT a drop target — renders without `useDroppable`, no highlight

### Post-Drop Animation

CSS transition on card arrival at destination: `transition: transform 100ms ease-out`.
Applied via `DragOverlay dropAnimation={{ duration: 150, easing: 'ease-out' }}` (dnd-kit built-in).
No custom animation needed beyond dnd-kit's built-in drop animation.

Source: CONTEXT.md D-17, RESEARCH.md Pattern 2.

### Server-Update Freeze During Drag

Not a visual contract but affects what the user sees. Incoming `STATE_UPDATE` messages are buffered while `isDraggingRef.current === true`. Applied on drag end. User does not see card teleport mid-drag.

Source: CONTEXT.md D-13, RESEARCH.md Pattern 1.

### Snap-Back on Failed Drop

If drop lands on invalid target (`over === null`) or server returns error, card animates back to original position using dnd-kit's drop animation at the source position. No custom snap animation needed.

## Layout Contract

### Board Regions (top to bottom, full viewport height)

```
┌─────────────────────────────────────┐
│  Opponent Strip  (~88px fixed)       │
│  One row of face-down card backs     │
│  per opponent, labeled "Player"      │
├─────────────────────────────────────┤
│  Pile Zone Row  (flex-1, centered)   │
│  Draw | Discard | Play Area          │
│  Horizontal row, centered vertically │
├─────────────────────────────────────┤
│  Player Hand Strip  (128px fixed)    │
│  Scrollable row of face-up cards     │
└─────────────────────────────────────┘
```

- Board fills full viewport: `h-screen w-screen overflow-hidden`
- Opponent strip: `h-[88px] flex items-center px-4 gap-2 bg-card`
- Pile row: `flex-1 flex items-center justify-center gap-6 px-4`
- Hand strip: `h-[128px] flex items-center px-4 gap-2 overflow-x-auto bg-card`

Source: CONTEXT.md D-05, D-06, D-07, specifics section (80-100px opponent, 120-140px hand). Using 88px and 128px as the specific values within those ranges.

### Pile Zone Visual

Each pile zone is a fixed-size container with:
- Dimensions: 80px wide × 112px tall (slightly larger than card to create a "slot" feel)
- Background: `bg-secondary` (hsl 160 30% 22%)
- Border: `border border-border rounded-lg` (default state)
- Drop-over border: `border-primary` (amber) — applied via `isOver` from `useDroppable`
- Pile name: 12px label above the zone container, weight 400, `text-muted-foreground`
- Card count badge: shadcn Badge, positioned bottom-right of zone, shows count
- Empty state: zone shows name + "0" badge + dashed border `border-dashed`
- Occupied state: shows top card face (or back if face-down), count badge

Source: CONTEXT.md D-07, D-09, D-11; RESEARCH.md Pattern 4.

### Card Dimensions

- Width: 63px, Height: 88px
- Border radius: `rounded-md` (from shadcn radius scale — `calc(var(--radius) * 0.8)` ≈ 8px)
- Card face border: `border border-gray-300`
- Card back border: `border border-gray-600`

Source: RESEARCH.md Pattern 4 (standard poker card ratio).

## Copywriting Contract

| Element | Copy |
|---------|------|
| Opponent label | "Player" |
| Pile names | "Draw" / "Discard" / "Play Area" (derived from server `pile.name` field) |
| Empty pile state | (no text label — empty zone visible via dashed border + "0" badge) |
| Empty hand state | (no text label — empty scrollable row, drop target remains active) |
| Drag cancel (no drop) | (silent — card snaps back, no toast) |
| Connection error | Handled by LobbyPanel (existing) — not a new Phase 3 state |
| Move error (snap-back) | (silent in Phase 3 — card snaps back, no error toast) |
| Card count format | "{N}" — plain integer, no "cards" suffix |

No primary CTA in Phase 3. The board is purely interactive — no buttons trigger named actions. Board transitions immediately from lobby when `gameState` is available (D-08).

No destructive actions in Phase 3. No confirmation dialogs needed.

Source: CONTEXT.md D-08, D-11; REQUIREMENTS.md TABLE-01, TABLE-02, TABLE-03.
