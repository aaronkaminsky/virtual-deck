In party/index.ts, replace the DRAW_CARD case block (lines 121-132) with explicit error checks before the happy path. The new logic:

```
case "DRAW_CARD": {
  if (!action.pileId) {
    sender.send(JSON.stringify({
      type: "ERROR",
      code: "MISSING_PILE_ID",
      message: "pileId is required for DRAW_CARD",
    } satisfies ServerEvent));
    break;
  }
  const pile = this.gameState.piles.find(p => p.id === action.pileId);
  if (!pile) {
    sender.send(JSON.stringify({
      type: "ERROR",
      code: "PILE_NOT_FOUND",
      message: `No pile found with id: ${action.pileId}`,
    } satisfies ServerEvent));
    break;
  }
  if (pile.cards.length === 0) {
    sender.send(JSON.stringify({
      type: "ERROR",
      code: "PILE_EMPTY",
      message: `Pile ${action.pileId} has no cards`,
    } satisfies ServerEvent));
    break;
  }
  const playerToken = sender.id;
  const card = pile.cards.pop()!;
  if (!this.gameState.hands[playerToken]) {
    this.gameState.hands[playerToken] = [];
  }
  this.gameState.hands[playerToken].push(card);
  break;
}
```

Do NOT change the `satisfies ServerEvent` pattern -- it ensures type safety.
Do NOT add a default pileId anywhere.
Do NOT modify any other action handlers.

Note: The `!action.pileId` check handles both undefined and empty string. The TypeScript type says pileId is `string`, but at runtime a malformed message cast via `as ClientAction` could have it missing.
