  <name>Task 1: Fix reset button gating (999.51)</name>
  <files>src/components/ControlsBar.tsx</files>
  <action>
    In `ControlsBar.tsx` line 28, replace:

      const resetDisabled = gameState.phase !== 'playing';

    with:

      const resetDisabled = !gameState.canUndo;

    Rationale: `canUndo` is `true` whenever `undoSnapshots.length > 0`. Every meaningful mutation (DEAL_CARDS, MOVE_CARD, FLIP_CARD, SHUFFLE, PLAY_CARD_SET, REORDER_PILE_SPREAD, SET_PILE_FACE) calls `takeSnapshot` before mutating. RESET_TABLE clears `undoSnapshots` and sets phase back to 'setup', which is exactly the "nothing to reset" state. The old `phase !== 'playing'` check disabled reset immediately after the user clicked Reset (phase → 'setup', no snapshot), but would also have disabled it in any future mutation before a deal — both wrong. No other changes needed.
  </action>
  <verify>
    <automated>npm run typecheck</automated>
  </verify>
  <done>Reset button is enabled after any deal, move, flip, or shuffle; disabled after a reset or in fresh lobby state.</done>
</task>
