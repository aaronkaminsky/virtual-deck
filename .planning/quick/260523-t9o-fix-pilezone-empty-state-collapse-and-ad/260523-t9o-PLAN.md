<objective>
Fix two visual regressions found during phase 31 GAP verification:

1. **PileZone empty-state collapse** (regression from plan 31-07): Empty PileZone droppable slots collapse to ~16px (just `py-2`) because plan 31-07 removed fixed `h-[64px] sm:h-[88px]` heights to fix GAP-04. The fix worked for populated slots but left empty slots without intrinsic content to size against. Solution: use `min-h-[64px] sm:min-h-[88px]` instead — preserves visual size when empty while still allowing intrinsic sizing when populated.

2. **Board needs min-height** (analogous to GAP-05 min-width): GAP-05 added `min-w-[320px]` to the board root. The board needs a matching `min-h-[480px]` so the layout does not collapse to zero/very-small height on short viewports.

Purpose: Restore consistent visual sizing for empty pile slots without re-introducing the GAP-04 fixed-height bug; harden the board against tiny viewport collapse.
Output: Updated PileZone.tsx and BoardView.tsx with `min-h` (not `h`) constraints.
</objective>
